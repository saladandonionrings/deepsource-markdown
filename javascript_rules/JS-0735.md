# Prevent DOM lookup in template `onCreated` callback
**ID:** `JS-0735` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0735)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

When the `onCreated` lifecycle callback is called, the template does not yet exist in the DOM. Trying to access its elements is most likely an error.

Callbacks added with this method are called before your template’s logic is evaluated for the first time. Inside a callback, `this` is the new template instance object. Properties you set on this object will be visible from the callbacks added with `onRendered` and `onDestroyed` methods and from event handlers.


## Bad Practice

```javascript
Template.foo.onCreated(function () {
  $('.bar').focus()
})

Template.foo.onCreated(function () {
  Template.instance().$('.bar').focus()
})
```

## Recommended

```javascript
Template.foo.onCreated(function () {
  console.log('hello')
})


Template.foo.onRendered(function () {
  $('.bar').focus()
  Template.instance().$('.bar').focus()
})

// should be a warning, but is too hard to check for statically,
// so the rule ignores it
Template.foo.onCreated(function () {
  this.$('.bar').focus()
})
```
