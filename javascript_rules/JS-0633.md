# Prefer having a valid `v-for` directive
**ID:** `JS-0633` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0633)

![Major](https://img.shields.io/badge/severity-major-orange)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

It is recommended to have valid `v-for` directives to avoid unexpected errors. We consider the following `v-for` directive invalid:

* The directive that has argument.* The directive that has modifier.* The directive that does not have attribute value.* If the element which has the directive is a custom component, the component does not have`v-bind:key`directive.* The`v-bind:key`directive does not use the variables which are defined by the`v-for`directive.

## Bad Practice

```javascript
<div v-for:todos></div>
<your-component v-for="item in list"></your-component>
<div v-for="x in list" :key="foo"></div>
```

## Recommended

```javascript
<div v-for="todo in todos"/>
<your-component
    v-for="item in list"
    :key="item.id"
/></your-component>
<div
    v-for="x in list"
    :key="x.id"
 />
```
