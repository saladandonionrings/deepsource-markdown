# Found usage of deprecated `get/getProperties` on Ember Objects
**ID:** `JS-0786` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0786)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Starting in Ember 3.1, native ES5 getters are available, which eliminates much of the need to use `get` / `getProperties` on Ember objects.

**Migration approaches to native methods**

* Replace`this.get('someProperty')`with`this.someProperty`.* Replace`{ prop1: this.prop1, prop2: this.prop2 }`with`this.getProperties('prop1', 'prop2')`.
But there are some cases where `get`/`getProperties` are valid:

* Ember proxy objects (`ObjectProxy`,`ArrayProxy`)* Objects implementing the`unknownProperty`method

## Bad Practice

```javascript
// Example 1
const property = this.get('someProperty');

// Example 2
import { get } from '@ember/object';
const prop = get(this, 'someProperty');

// Example 3
const { prop1, prop2 } = this.getProperties('prop1', 'prop2');

// Example 4
import { getProperties } from '@ember/object';
const props = getProperties(this, 'prop1', 'prop2');
```

## Recommended

```javascript
// Example 1
const property = this.someProperty;

// Example 2
// Optional chaining can be useful if the nested path can have null or undefined properties in it.
const foo = this.nested?.path; 

// Example 3
const { prop1, prop2 } = this;

// Example 4
import ObjectProxy from '@ember/object/proxy';

export default ObjectProxy.extend({
  someFunction() {
    const foo = this.get('propertyInsideProxyObject'); // Allowed because inside proxy object.
  }
});
```
