# Found invalid `v-slot` directives
**ID:** `JS-0641` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0641)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

`v-slot` denotes named slots or slots that expect to receive props. It is limited to `<template>` and components (for a lone default slot with props).

`v-slot` directive is invalid when:

* The directive is not owned by a custom element.
* The directive is a named slot and is on a custom element directly.
* The directive is the default slot, is on a custom element directly, and there are other named slots.
* The element which has the directive has another `v-slot` directive.
* The element which has the directive has another `v-slot` directive that is distributed to the same slot.
* The directive has a dynamic argument which uses the scope properties that the directive defined.
* The directive is the default slot, is on a custom element directly, and has no value.
* The directive has any modifier.


## Bad Practice

```javascript
<!-- 'v-slot' directive must be owned by a custom element, but 'div' is not -->
<div v-slot="data">
  {{data}}
</div>

<!-- Named slots must use '<template>' on a custom element -->
<my-component v-slot:one="data">
  {{data}}
</my-component>

<!-- Default slot must use '<template>' on a custom element when there are other named slots -->
<my-component v-slot="data">
  {{data}}
  <template v-slot:one>
    one
  </template>
</my-component>

<!-- An element cannot have multiple 'v-slot' directives -->
<my-component v-slot:one v-slot:two>
  one and two
</my-component>

<!-- An element cannot have multiple '<template>' elements which are distributed to the same slot -->
<my-component>
  <template v-slot:one>
    one 1
  </template>
  <template v-slot:one>
    one 2
  </template>
</my-component>

<!-- Dynamic argument of 'v-slot' directive cannot use that slot parameter -->
<my-component>
  <template v-slot:[data]="data">
    dynamic?
  </template>
</my-component>

<!-- 'v-slot' directive doesn't support any modifier -->
<my-component v-slot.mod="data">
  {{data}}
</my-component>

<!-- 'v-slot' directive on a custom element requires that attribute value -->
<my-component v-slot>
  content
</my-component>
```

## Recommended

```javascript
<!-- 'v-slot' directive must be owned by a custom element, but 'div' is not -->
<my-comp>
  <template v-slot="data">
    {{data}}
  </template>
</my-comp>

<!-- Named slots must use '<template>' on a custom element -->
<my-component>
  <template v-slot:one="data">
    {{data}}
  </template>
</my-component>

<!-- Default slot must use '<template>' on a custom element when there are other named slots -->
<!-- Fix by using template for slot instead of defining directly in custom component -->
<my-component>
  <template v-slot="data">
    {{data}}
  </template>
  <template v-slot:one>
    one
  </template>
</my-component>

<!-- An element cannot have multiple 'v-slot' directives -->
<!-- Fix by using one `v-slot` directive per template -->
<my-component>
  <template v-slot:one>
    one
  </template>
  <template v-slot:two>
    two
  </template>
</my-component>

<!-- An element cannot have multiple '<template>' elements which are distributed to the same slot -->
<!-- Fix by renaming the same slot with another name here, it was `v-slot:two` -->
<my-component>
  <template v-slot:one>
    one 1
  </template>
  <template v-slot:two>
    one 2
  </template>
</my-component>

<!-- Dynamic argument of 'v-slot' directive cannot use that slot parameter -->
<!-- Fix by renaming the dynamic slot to static slot -->
<my-component>
  <template v-slot:staticslot="data">
    static slot
  </template>
</my-component>

<!-- 'v-slot' directive doesn't support any modifier -->
<!-- Here it is not a named slot so we can define in custom component -->
<my-component v-slot="data">
  {{data}}
</my-component>

<!-- 'v-slot' directive on a custom element requires an attribute value -->
<!-- Fix by adding a attribute value -->
<my-component v-slot="data">
  content
</my-component>
```
