# Found invalid `v-model` directives
**ID:** `JS-0636` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0636)

![Major](https://img.shields.io/badge/severity-major-orange)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

`v-model` directive is not valid :

* The directive used on`HTMLElement`has an argument.* The directive used on`HTMLElement`has modifiers which are not supported.* The directive does not have that attribute value.* The directive does not have the attribute value which is valid as Left-Hand Side.* The directive has potential null object property access.* The directive is on unsupported elements* The directive is on`<input>`elements which their types are file.* The directive's reference is iteration variables.

## Bad Practice

```javascript
<!-- 'v-model' directives require an attribute value -->
<input v-model>

<!-- 'v-model' directives require no argument -->
<input v-model:aaa="foo">

<!-- 'v-model' directives doesn't support modifiers -->
<input v-model.bbb="foo">

<!-- 'v-model' directives require the attribute value which is valid as LHS. -->
<input v-model="foo + bar">

<!-- 'v-model' directives aren't supported on <div> elements -->
<div v-model="foo"/>

<!-- 'v-model' directives cannot update the iteration variable 'todo' itself. -->
<div v-for="todo in todos">
    <input v-model="todo">
</div>
```

## Recommended

```javascript
<!-- 'v-model' directives require an attribute value -->
<input v-model="foo">

<!-- 'v-model' directives require no argument -->
<input v-model="foo">

<!-- 'v-model' directives doesn't support modifiers -->
<input v-model.lazy="foo">

<!-- 'v-model' directives require the attribute value which is valid as LHS. -->
<input v-model="foo">

<!-- 'v-model' directives aren't supported on <div> elements -->
<textarea v-model="foo"/>

<!-- 'v-model' directives cannot update the iteration variable 'todo' itself. -->
<div v-for="todo in todos">
  <input v-model="todo.name">
</div>
```
