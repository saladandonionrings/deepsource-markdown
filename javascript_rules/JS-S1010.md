# Found code vulnerable to shell command injection
**ID:** `JS-S1010` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-S1010)

![Critical](https://img.shields.io/badge/severity-critical-red)![Security](https://img.shields.io/badge/type-security-red)

The `execa.command` function is capable of spawning processes and therefore should be used with caution. Closely inspect all arguments passed to this function and ensure that they're either not being influenced by user input, or being sanitized before execution. Alternatively, call `execa` directly as a function and pass the argument list as an array of strings.

A [shell argument injection attack](https://owasp.org/www-community/attacks/Command_Injection) occurs when an attacker is able to execute arbitrary shell commands on a server. To prevent such attacks, sanitize all input passed to functions calls that are capable of executing shell commands.


## Bad Practice

```javascript
import express from 'express'
import execa from 'execa'

const app = express()
app.get('/ls/:flag', async (req, res) => {
  const result = await execa.command("ls" + req.params.flag) // vulnerable
  res.json(result)
})
```

## Recommended

```javascript
// Avoid passing user controlled arguments to `execa.command`:
const result = await execa.command("ls -l") // literal
const result2 = await execa.command(foo) // 'foo' is not passed by user 


// ALTERNATIVE: skipcq comment + sanitizater function.
// We can use a sanitizer function to validate the input and then pass it to
// `execa.command`. Additionally, it is recommended to add a skipcq comment to stop 
// this issue from being raised and clarify that the code is indeed safe.

import express from 'express'
import execa from 'execa'

const app = express()
app.get('/exec/:command', async (req, res) => {
  const sanitized = sanitize(command) // some sanitization function
  const result = await execa.command(req.params.command) // skipcq: JS-S1010 input has been validated 
  res.json(result)
})
```
