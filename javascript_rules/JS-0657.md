# Deprecation of `Vue.config.keyCodes`
**ID:** `JS-0657` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0657)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

In Vue 3.0+, using keyCodes as `v-on` modifiers is no longer supported. Following this change, `Vue.config.keyCodes` has been deprecated.

In Vue 2, keyCodes were supported as a way to modify a v-on method.


## Bad Practice
In the template:


```javascript
<!-- keyCode version -->
<input v-on:keyup.112="showHelpText" />

<!-- custom alias version -->
<input v-on:keyup.f1="showHelpText" />
```
In the Vue script:


```javascript
Vue.config.keyCodes = {
  f1: 112
}
```

## Recommended
It is now recommended to use the kebab-case name for any key you want to use as a modifier.


```javascript
<template>
  <div>
    <!-- Matches both q and Q -->
    <input v-on:keypress.q="quit">
    <!-- Vue 3 Key Modifier on v-on -->
    <input v-on:keyup.page-down="nextPage">
  </div>
</template>
```
