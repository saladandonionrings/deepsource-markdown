# Using `fetch()` without importing module
**ID:** `JS-W1006` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-W1006)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

The issue is raised when it founds the `fetch()` calls with no corresponding import declaration is found.

The issue can be fixed by adding `import fetch from 'fetch';` to all files that need it.

Using `fetch()` without importing it causes the browser to use the native, non-wrapped `window.fetch()`. This is generally fine, but makes testing harder because this non-wrapped version does not have a built-in test waiter. Due to which, it is generally better to use ember-fetch and explicitly `import fetch from 'fetch'`.


## Bad Practice

```javascript
const result = fetch('/something');
```

## Recommended

```javascript
import fetch from 'fetch';

const result = fetch('/something');
```
