# Disallows usage of forwardRef
**ID:** `JS-0583` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0583)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

`forwardRef` is used when the `token` which we need to refer to for the purposes of Dependency Injection(DI) is declared, but not yet defined. It is also used when the `token` used during creation of a query is not yet defined.

The flow of DI is disrupted by using forwardRef, and might make code more difficult to understand.


## Bad Practice

```javascript
const TAGS_VALUE_ACCESSOR: StaticProvider = {
    multi: true,
    provide: NG_VALUE_ACCESSOR,
    useExisting: forwardRef(() => TagsValueAccessor)
};
@Directive({
    providers: [TAGS_VALUE_ACCESSOR],
    selector: '[tags]'
})
export class TagsValueAccessor {}
```

## Recommended

```javascript
@Component({
    selector: 'test',
    template: ''
})
export class Test {
    constructor() {
    this.test();
    }
    test() {}
}
```
