# Deprecation of functional template
**ID:** `JS-0648` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0648)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Following the release of Vue 3.0, the following changes have been made:

* Functional components must be written as plain functions. `{ functional: true }` option removed. `<template functional>` no longer supported.
* `{ functional: true }` option removed.
* `<template functional>` no longer supported.
* Async components must now be created via a dedicated API method.

* `{ functional: true }` option removed.
* `<template functional>` no longer supported.


## Bad Practice

```javascript
const FunctionalComp = {
  functional: true,
  render(h) {
    return h('div', `Hello! ${props.name}`)
  }
}

<template functional></template>
```

## Recommended

```javascript
import { h } from 'vue'
const FunctionalComp = props => {
return h('div', `Hello! ${props.name}`)
}

import { defineAsyncComponent } from 'vue'
const AsyncComp = defineAsyncComponent(() => import('./Foo.vue'))
```
