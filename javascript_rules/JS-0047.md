# No default cases in switch statements
**ID:** `JS-0047` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0047)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Some code conventions require that all switch statements have a default case, even if the default case is empty. The idea is to always explicitly state what the default behavior should be so that it's clear that the developer didn't forget to handle the default fallback case.


## Bad Practice

```javascript
switch (a) {
    case 1:
        /* code */
        break;
}
```

## Recommended

```javascript
switch (a) {
    case 1:
        /* code */
        break;

    default:
        /* code */
        break;
}

switch (a) {
    case 1:
        /* code */
        break;

    // no default
}

switch (a) {
    case 1:
        /* code */
        break;

    // No Default
}
```
