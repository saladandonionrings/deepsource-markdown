# Should use snake case for dynamic route segments
**ID:** `JS-0816` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0816)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Dynamic segments in routes should use snake case, so Ember doesn't have to do extra serialization in order to resolve promises.


## Bad Practice

```javascript
this.route('tree', { path: ':treeId' });
```

## Recommended

```javascript
this.route('tree', { path: ':tree_id' });
```
