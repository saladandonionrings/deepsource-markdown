# Should not use global `Session`
**ID:** `JS-0727` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0727)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

`Session` variables live in a global namespace, which is bad practice. `ReactiveDict` should be used instead.


## Bad Practice

```javascript
Session.set('foo')
Session.get('foo')
Session.all()
Session.clear()
```

## Recommended

```javascript
ReactiveDict.get('foo')
ReactiveDict.set('foo', 'bar')
ReactiveDict.setDefault('foo', 'bar')
ReactiveDict.equals('foo', 'bar')
ReactiveDict.all()
ReactiveDict.clear()
ReactiveDict.destroy()
```
