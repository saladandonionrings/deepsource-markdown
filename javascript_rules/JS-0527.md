# Found complex run functions
**ID:** `JS-0527` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0527)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Any code that needs to run when an application starts should be declared in a factory, exposed via a function, and injected into the run block.

* Code placed directly in a run block can be difficult to test. Placing it in a factory makes it easier to abstract and mock.
* Code placed directly in a run block can cause race conditions for startup logic, as it does not have a way to communicate when asynchronous code in the run block has completed.


## Bad Practice

```javascript
angular
    .module('app')
    .run(function(KITTENS) {
        $window.addEventListener('deviceready', prefetchData);
        function prefetchData(KITTENS) {
          // do something
          startup();
        }

        function startup(){
           // do something
        }
    });
```

## Recommended

```javascript
angular
    .module('app')
    .run(function(KITTENS, kittenService, startup) {
        kittenService.prefetchData(KITTENS);
        startup();
    });
```
