# Should not use `.on()`
**ID:** `JS-0800` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0800)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Avoid using `.on()` in favour of component's lifecycle hooks. The order of execution for `on()` is not deterministic.


## Bad Practice

```javascript
export default Component.extend({
  abc: on('didInsertElement', function () {
    /* custom logic */
  })
```

## Recommended

```javascript
export default Component.extend({
  didInsertElement() {
    /* custom logic */
  }
});
```
