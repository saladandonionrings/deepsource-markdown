# Unnecessary `catch` clauses found
**ID:** `JS-0112` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0112)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Performance](https://img.shields.io/badge/type-performance-white)

A `catch` clause that only rethrows the original error is redundant, and has no effect on the runtime behavior of the program. These redundant clauses can be a source of confusion and code bloat, so it's better to disallow these unnecessary catch clauses.


## Bad Practice

```javascript
try {
  doSomethingThatMightThrow();
} catch (e) {
  throw e;
}

try {
  doSomethingThatMightThrow();
} catch (e) {
  throw e;
} finally {
  cleanUp();
}
```

## Recommended

```javascript
try {
  doSomethingThatMightThrow();
} catch (e) {
  doSomethingBeforeRethrow();
  throw e;
}

try {
  doSomethingThatMightThrow();
} catch (e) {
  handleError(e);
}

try {
  doSomethingThatMightThrow();
} finally {
  cleanUp();
}
```
