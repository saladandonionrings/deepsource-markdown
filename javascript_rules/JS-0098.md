# Void operators found
**ID:** `JS-0098` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0098)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

The void operator takes an operand and returns undefined. It can be used to ignore the value produced by an expression. However, this can lead to code that is difficult to understand and maintain. Historically, the void operator was used to get a "pure" undefined value, as the undefined variable was mutable prior to ES5. Nowadays, this is no longer necessary as undefined is immutable in modern JavaScript.


## Bad Practice

```javascript
// will always return undefined
(function(){
    return void 0;
})();
```

## Recommended

```javascript
// Use explicit undefined
(function(){
    return undefined;
})();
```
