# Use default imports for only default exports
**ID:** `JS-W1028` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-W1028)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

It is recommended to use default imports only if there is a default export in the module being imported. If there are no default exports in the source module, it is better to use named imports, and it helps the build tools while optimizing the code.

Default exports are mainly used to maintain a single file for a particular module. This helps in code navigation and code refactoring as well. It is also common to have a single big module export as default and related modules exported through named exports in a single file. These default exports are imported through default import. That is, they are exported using the keyword name `default` .


```javascript
// ./handler/card.js

export default class Card {}
```
and this can be imported using default import


```javascript
// index.js

import CardHandler from './handler/card.js'
```

## Bad Practice

```javascript
// ./handler/card.js
export class Card {}
```

```javascript
// index.js
import Card from './handler/card.js'
```

## Recommended

```javascript
// ./handler/card.js
export class Card {}
```

```javascript
// index.js
import { Card } from './handler/card.js'
```
