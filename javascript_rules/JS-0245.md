# Require spread operators instead of .apply()
**ID:** `JS-0245` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0245)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Before ES2015, one must use `Function.prototype.apply()` to call variadic functions.


```javascript
var args = [1, 2, 3, 4];
Math.max.apply(Math, args);
```
In ES2015, one can use spread syntax to call variadic functions.


```javascript
var args = [1, 2, 3, 4];
Math.max(...args);
```

## Bad Practice

```javascript
foo.apply(undefined, args);
foo.apply(null, args);
obj.foo.apply(obj, args);
```

## Recommended

```javascript
// Using spread syntax
foo(...args);
obj.foo(...args);

// The `this` binding is different.
foo.apply(obj, args);
obj.foo.apply(null, args);
obj.foo.apply(otherObj, args);

// The argument list is not variadic.
// Those are warned by the `no-useless-call` rule.
foo.apply(undefined, [1, 2, 3]);
foo.apply(null, [1, 2, 3]);
obj.foo.apply(obj, [1, 2, 3]);
```
