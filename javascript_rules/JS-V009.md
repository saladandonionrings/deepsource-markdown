# Prefer the use of `===` and `!==` over `==` and `!=`
**ID:** `JS-V009` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-V009)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

It is considered good practice to use the type-safe equality operators `===` and `!==` instead of their regular counterparts `==` and `!=` . For instance, the following statements are all considered true:


## Bad Practice

```javascript
a == b
c == true
bananas != 1
value == undefined
typeof c == 'undefined'
'hello' != 'world'
0 == 0
true == true
c == null
```

## Recommended

```javascript
a === b
c === true
bananas !== 1
value === undefined
typeof c === 'undefined'
'hello' !== 'world'
0 === 0
true === true
c === null
```
