# Found duplicate assignments
**ID:** `JS-W1032` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-W1032)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Consecutively reassigning to the same variable or property is a code smell and should be avoided.

It is likely that this is the result of some undeleted code and can have unexpected side effects. The first assignment is rendered useless by the second and can therefore be removed without observing any changes to the value of the assignment target.


## Bad Practice

```javascript
a[1] = 'something'
a[1] = 'some other thing'
```

## Recommended

```javascript
a[1] = 'something'
if (condition()) {
  a[1] = 'some other thing'
}
```
