# Audit: Found insecure randomness in initialization of sensitive data
**ID:** `JS-A1000` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-A1000)

![Critical](https://img.shields.io/badge/severity-critical-red)![Security](https://img.shields.io/badge/type-security-red)

Numbers generated by `Math.random` are not cryptographically secure. When using random numbers in security sensitive contexts, it is recommended to go with cryptographically secure sources of randomness.

In the browser, you can use [Crypto.getRandomValues](https://developer.mozilla.org/en-US/docs/Web/API/Crypto/getRandomValues). For NodeJS environments, you can use the built-in [getRandomValues](https://nodejs.org/api/crypto.html#cryptogetrandomvaluestypedarray) from the "crypto" module.


## Bad Practice

```javascript
// In NodeJS
const password = `${basePassword}-${Math.random() * 1000}`

// In the browser
const formData = getUserInput()
formData.password += Math.random()
```

## Recommended

```javascript
// In NodeJS
const crypto = require('crypto')
const suffix = crypto.randomBytes(seed)[0]
const password = `${basePassword}-${suffix}`

// In the browser
const formData = getUserInput()
formData.password += window.crypto.getRandomValues(seed)[0]
```
