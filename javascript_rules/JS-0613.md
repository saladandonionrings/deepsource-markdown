# Avoid overwriting reserved keys
**ID:** `JS-0613` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0613)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Use of reserved names should be prevented to avoid conflicts and unexpected behavior. This issue can be fixed by renaming the keys.


## Bad Practice

```javascript
<script>
export default {
  props: {
    // $el is reserved key
    $el: String
  },
  computed: {
    // $on is reserved key
    $on: {
      get () {}
    }
  },
  data: {
      // keys starting with with '_' are reserved
    _property: null
  },
  methods: {
    // $nextTick is reserved key
    $nextTick () {}
  }
}
</script>
```

## Recommended

```javascript
<script>
export default {
  props: {
    $elKey: String
  },
  computed: {
    $onKey: {
      get () {}
    }
  },
  data: {
    property: null
  },
  methods: {
    $nextTickKey () {}
  }
}
</script>
```
