# Detected using a non-null assertion after an optional chain expression
**ID:** `JS-0338` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0338)

![Major](https://img.shields.io/badge/severity-major-orange) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Optional chain expressions are designed to return `undefined` if the optional property is nullish. Using non-null assertions after an optional chain expression is wrong, and introduces a serious type safety hole into your code.


## Bad Practice

```javascript
foo?.bar!;
foo?.bar!.baz;
foo?.bar()!;
foo?.bar!();
foo?.bar!().baz;
```

## Recommended

```javascript
foo?.bar;
(foo?.bar).baz;
foo?.bar();
foo?.bar();
foo?.bar().baz;
```
