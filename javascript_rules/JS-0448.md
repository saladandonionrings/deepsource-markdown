# Avoid usage of `shouldComponentUpdate`
**ID:** `JS-0448` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0448)

![Major](https://img.shields.io/badge/severity-major-orange)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

`PureComponent` changes the life-cycle method `shouldComponentUpdate` and adds some logic to automatically check whether a re-render is required for the component.

This allows a `PureComponent` to call method to render only if it detects changes in state or props, hence, one can change the state in many components without having to write extra checks.

Use `shouldComponentUpdate()` to let React know if a component’s output is not affected by the current change in state or props. The default behavior is to re-render on every state change, and in the vast majority of cases you should rely on the default behavior.

**Recommended Solutions**


## Bad Practice

```javascript
class avoidComponentClass extends React.PureComponent {
  shouldComponentUpdate() {
    // do check
  }

  render() {
    return <div>Radical!</div>
  }
}

function avoidPureComponent() {
  return class avoidPureComponentClass extends React.PureComponent {
    shouldComponentUpdate() {
      // do check
    }

    render() {
      return <div>Groovy!</div>
    }
  }
}
```

## Recommended

```javascript
class avoidComponentClass extends React.Component {
  shouldComponentUpdate() {
    // do check
  }

  render() {
    return <div>Radical!</div>
  }
}

function avoidPureComponent() {
  return class avoidPureComponentClass extends React.Component {
    shouldComponentUpdate() {
      // do check
    }

    render() {
      return <div>Groovy!</div>
    }
  }
}
```
