# Found `key` attribute on `<template v-for>`
**ID:** `JS-V001` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-V001)

![Critical](https://img.shields.io/badge/severity-critical-red)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Do not use `:key` because `<template>` is not an element in the virtualDOM. This is a helper syntax of sorts, a wrapper to encapsulate elements, it does not render itself. It just renders what's inside it. So developers are required to uniquely assign `:key` to the elements inside the template.


## Bad Practice

```javascript
<template>
  <template v-for="item in list" :key="item.id">
    <div />
  </template>
</template>
```

## Recommended

```javascript
<template>
  <template v-for="item in list">
    <div :key="item.id" />
  </template>
</template>
```
