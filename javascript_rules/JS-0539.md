# Prefer having a prefix for all filter names
**ID:** `JS-0539` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0539)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

It is recommended to specify a consistent preference for naming filters. This provides a consistent way to identify and reference filters quickly. The issue is recommended for the Angular versions v1.x.

Note: This issue is raised only when the project has an existing ESLint configuration file.


## Bad Practice

```javascript
// error: The otherFilter filter should follow this pattern: /^xyz/
angular
    .module('myModule')
    .filter('otherFilter', function () {
        // ...
    });
```

## Recommended

```javascript
angular
    .module('myModule')
    .filter('prefixFilter', function () {
        // ...
    });
```
