# Duplication of field names is not allowed
**ID:** `JS-0608` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0608)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Multiple properties with the same key in object literals can cause unexpected behavior in your application.


## Bad Practice

```javascript
<script>
export default {
  props: {
    property: String
  },
  computed: {
    property: {
      get () {}
    }
  },
  data: {
    property: null
  },
  methods: {
    property () {}
  }
}
</script>
```

## Recommended

```javascript
<script>
export default {
  props: {
    property: String
  },
  computed: {
    computedProperty: {
      get () {}
    }
  },
  data: {
    propName: null
  },
  methods: {
    user () {}
  }
}
</script>
```
