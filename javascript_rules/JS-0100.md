# `with` statements found
**ID:** `JS-0100` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0100)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

The `with` statement is potentially problematic because it adds members of an object to the current scope, making it impossible to tell what a variable inside the block actually refers to.


## Bad Practice

```javascript
with (point) {
    r = Math.sqrt(x * x + y * y); // is r a member of point?
}
```

## Recommended

```javascript
const r = ({x, y}) => Math.sqrt(x * x + y * y);
```
