# Unnecessary concatenation of literals or template literals found
**ID:** `JS-0096` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0096)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

It's unnecessary to concatenate two string literals together, such as:


```javascript
let foo = "a" + "b";
```
This code is likely the result of refactoring where a variable was removed from the concatenation (such as "a" + b + "b"). In such a case, the concatenation isn't important and the code can be rewritten as:


```javascript
let foo = "ab";
```

## Bad Practice

```javascript
let a = `some` + `string`;

// these are the same as "10"
a = '1' + '0';
a = '1' + `0`;
a = `1` + '0';
a = `1` + `0`;
```

## Recommended

```javascript
// when a non string is included
let c = a + b;
c = '1' + a;
let a = 1 + '1';
c = 1 - 2;
// when the string concatenation is multiline
c = "foo" + "bar";
```
