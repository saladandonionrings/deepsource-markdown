# Found deprecated Ember string prototype extensive methods
**ID:** `JS-W1003` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-W1003)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

The issue is raised when it found the `Ember.String` or String prototypes extensive methods.

The issue can be fixed as follows:

* Use`@ember/string`, which replaces`Ember.String`* Use`ember-string-prototype-extensions`, which brings String prototype extensions back
Much of the public API of Ember was designed and published some time ago, when the client-side landscape looked much different. It was a time without many utilities and methods that have been introduced to JavaScript since, without the current rich npm ecosystem, and without ES6 modules. On the Ember side, Ember CLI and the subsequent addons were still to be introduced. Global mode was the way to go, and extending native prototypes like Ember does for `String`, `Array` and `Function` was a common practice.

After Ember 2.x:

* String protoype extensions are deprecated* `Ember.String`functions are deprecated* `loc`is completely deprecated* `isHTMLSafe`and`htmlSafe`are moved to`@ember/template`

## Bad Practice

```javascript
'myString'.dasherize();
someString.capitalize();
'<b>foo</b>'.htmlSafe();
```

## Recommended

```javascript
dasherize('myString');
capitalize(someString);
htmlSafe('<b>foo</b>');
```
