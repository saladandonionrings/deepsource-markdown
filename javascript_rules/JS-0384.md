# Bad usage of triple slash directives
**ID:** `JS-0384` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0384)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Use of triple-slash reference type directives is discouraged in favor of the newer `import` style. This rule allows you to ban use of `/// <reference path="" />` , `/// <reference types="" />` , or `/// <reference lib="" />` directives.


## Bad Practice

```javascript
/// <reference types="foo" />
import * as foo from 'foo';

/// <reference types="foo" />
import foo = require('foo');
```

## Recommended

```javascript
import * as foo from 'foo';

import * as foo from 'foo';
```
