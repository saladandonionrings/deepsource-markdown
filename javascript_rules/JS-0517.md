# Use one component per file
**ID:** `JS-0517` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0517)

![Major](https://img.shields.io/badge/severity-major-orange) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

The number of AngularJS components per file should be limited to one because:

* It promotes easier unit testing and mocking.
* It makes it far easier to read, maintain, and avoid collisions with teams in source control.
* It avoids hidden bugs that often arise when combining components in a file where they may share variables, create unwanted closures, or unwanted coupling with dependencies.


## Bad Practice

```javascript
angular
    .module('app', ['ngRoute'])
    .controller('SomeController', SomeController)
    .factory('someFactory', someFactory);

function SomeController() { }

function someFactory() { }
```

## Recommended

```javascript
// app.module.js
angular
    .module('app', ['ngRoute']);

// some.controller.js
angular
    .module('app')
    .controller('SomeController', SomeController);

function SomeController() { }

// some.factory.js
angular
    .module('app')
    .factory('someFactory', someFactory);

function someFactory() { }
```
