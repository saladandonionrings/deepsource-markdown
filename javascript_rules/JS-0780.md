# Avoid using `@each` in deeply nested computed property
**ID:** `JS-0780` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0780)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Performance](https://img.shields.io/badge/type-performance-white)

Ember does not allow deeply-nested computed property dependent keys with `@each`. At runtime, it will show a warning about this.

You should use `@each` only one level deep otherwise it will create a complex equation and effect your application performance.


## Bad Practice

```javascript
export default Component.extend({
  displayNames: computed('[[email protected]](/cdn-cgi/l/email-protection)', function () {
    return this.todos.map((todo) => todo.owner.name);
  })
});
```

## Recommended

```javascript
export default Component.extend({
  displayNames: computed('[[email protected]](/cdn-cgi/l/email-protection)', function () {
    return this.owners.mapBy('name');
  }),

  owners: computed('[[email protected]](/cdn-cgi/l/email-protection)', function () {
    return this.todos.mapBy('owner');
  })
});
```
