# Prevent assigning modules to variables
**ID:** `JS-0515` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0515)

![Major](https://img.shields.io/badge/severity-major-orange) ![Style](https://img.shields.io/badge/type-style-blue)

It is a good practice to use one component per file. There is rarely a need to introduce a variable to a module. The developers should declare modules without variable declaration.


## Bad Practice

```javascript
var app = angular.module('app', [
    'ngAnimate',
    'ngRoute',
    'app.shared',
    'app.dashboard'
]);
```

## Recommended

```javascript
angular
    .module('app', [
        'ngAnimate',
        'ngRoute',
        'app.shared',
        'app.dashboard'
    ]);
```
