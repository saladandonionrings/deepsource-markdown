# Found non-existent assignment operators
**ID:** `JS-W1033` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-W1033)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

When using compound assignment operators, it is esay to mistake `=-` for `-=` , or `=+` for `+=` . The code will compile and execute, but won't behave as expected.

A code snippet like `x =- 1` is highly likely to have been a mistyped `x -= 1` or `x = -1` . DeepSource autofix will assume the latter variant and replace the erroneous code with an assignment unary expression.


## Bad Practice

```javascript
let x = make_x()
x =- something
```

```javascript
let x = make_x()
x =-1 // could be a typo of `x -= 1`
```

## Recommended

```javascript
let x = make_x()
x -= 1
// or
x = -1
```
