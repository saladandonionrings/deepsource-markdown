# Avoid using arrow functions to define watcher
**ID:** `JS-0606` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0606)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Watchers will not be able to reference the instance of `Vue` if defined as arrow functions.

Arrow functions do not retain the `this` from their parent context, and hence will not reference the Vue instance as one would expect. It is recommended to use a function expression instead.


## Bad Practice

```javascript
<script>
export default {
  watch: {
    watchProperty: (val, oldVal) => {
      console.log('new: %s, old: %s', val, oldVal)
    }
}
```

## Recommended

```javascript
<script>
export default {
  watch: {
    watchProperty: function (val, oldVal) {
      console.log('new: %s, old: %s', val, oldVal)
    },
}
</script>
```
