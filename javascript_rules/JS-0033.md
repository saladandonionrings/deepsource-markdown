# Found usage of comparison with negative-zero(`-0`)
**ID:** `JS-0033` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0033)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

JavaScript has two different representations for zero: positive zero, represented by `+0` (or just `0` ), and negative zero, represented by `-0` . The comparison with `-0` might be a typing error, for it would be clearer to compare against `0` . Trying to use equality operator ( `===` or `==` ) to compare against `-0` will not work as intended since code like `x === -0` will pass for both `+0` and `-0` .


```javascript
console.log(+0 === -0) // true
console.log(+0 == -0) // true
```
Try using `Object.is` instead which was introduced in ES2015 and differentiates between `-0` and `+0` .


## Bad Practice

```javascript
if (x === -0) {
    // doSomething()...
}
```

## Recommended

```javascript
if (x === 0) {
    // doSomething()...
}

//OR

if (Object.is(x, -0)) {
    // doSomething()...
}
```
