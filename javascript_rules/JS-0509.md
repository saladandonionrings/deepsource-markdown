# Prefer a consistent naming pattern for type aliases
**ID:** `JS-0509` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0509)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

`'^([A-Z][a-z0-9]*)+Type$$'` is the default pattern for type alias names.


## Bad Practice

```javascript
type foo = {};
```

## Recommended

```javascript
type FooType = {};
```
