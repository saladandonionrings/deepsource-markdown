# Props are not `read-only`
**ID:** `JS-0461` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0461)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Using Flow, one can define types for props. This issue enforces that prop types are read-only. Readonly is used to ensure that a property doesn't mutate over time.


## Bad Practice

```javascript
type Props = {
  name: string,
}

class Hello extends React.Component<Props> {
  render () {
    return <div>Hello {this.props.name}</div>;
  }
}

function Hello(props: {-name: string}) {
  return <div>Hello {props.name}</div>;
}

const Hello = (props: {|name: string|}) => (
  <div>Hello {props.name}</div>
);
```

## Recommended

```javascript
type Props = {
  name: string,
}
class Hello extends React.Component<Props> {
  render () {
    return <div>Hello {this.props.name}</div>;
  }
}

function Hello(props: {+name: string}) {
  return <div>Hello {props.name}</div>;
}

const Hello = (props: {|+name: string|}) => (
  <div>Hello {props.name}</div>
);
```
