# Require and specify a prefix for all provider names
**ID:** `JS-0541` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0541)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

All your providers should have a name starting with the parameter you can define in your config object. The second parameter can be a Regexp wrapped in quotes. You can not prefix your providers by "$" (reserved keyword for AngularJS services).

**Rule based on Angular 1.x**


## Bad Practice

```javascript
// valid
angular.module('myModule').provider('prefixProvider', function () {
    // ...
});
```

## Recommended

```javascript
// invalid
angular.module('myModule').provider('otherProvider', function () {
    // ...
}); // error: The otherProvider provider should follow this pattern: /^xyz/
```
