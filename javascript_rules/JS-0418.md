# Possible insertion of comments as text nodes
**ID:** `JS-0418` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0418)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Prevent comment strings (e.g. beginning with `//` or `/*` ) from being accidentally injected as a text node in JSX statements.

React will render the comments as text, so we recommend using comments in the JSX Expressions.


## Bad Practice

```javascript
// Example 1
var Hello = createReactClass({
  render: function() {
    return (
      <div>// empty div</div>
    );
  }
});

// Example 2
var Hello = createReactClass({
  render: function() {
    return (
      <div>
        /* empty div */
      </div>
    );
  }
});
```

## Recommended

```javascript
// Example 1
var Hello = createReactClass({
  displayName: 'Hello',
  render: function() {
    return <div>{/* empty div */}</div>;
  }
});

// Example 2
var Hello = createReactClass({
  displayName: 'Hello',
  render: function() {
    return <div /* empty div */></div>;
  }
});
```
