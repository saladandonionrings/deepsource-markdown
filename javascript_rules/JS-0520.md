# Avoid assignments to `$scope` in controllers
**ID:** `JS-0520` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0520)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

We should not set properties on `$scope` in controllers. Use the `controllerAs` syntax and add data to `this`. The `controllerAs` syntax uses `this` inside controllers which gets bound to `$scope`. `controllerAs` is syntactic sugar over `$scope`. You can still bind to the View and still access `$scope` methods. It helps avoid the temptation of using `$scope` methods inside a controller. It is recommended to avoid them or move the method to a factory, and reference them from the controller. The `$scope` is used in a controller when publishing and subscribing events using `$emit`, `$broadcast`, `$watch`, or `$on`.


## Bad Practice

```javascript
angular
    .module('myModule')
    .controller('SomeController', function($scope) {
        $scope.value = 42;
    });
```

## Recommended

```javascript
angular
    .module('myModule')
    .controller('SomeController', function($scope) {
        // `this` for values
        this.value = 42;

        // `$scope` is fine for watchers
        $scope.$watch(angular.bind(this, function () {
            return this.value
        }), function () {
            // ...
        });
    });
```
