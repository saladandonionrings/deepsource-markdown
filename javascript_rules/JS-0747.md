# Prefer to have content in heading elements
**ID:** `JS-0747` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0747)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Prefer that heading elements `(h1, h2, etc.)` have content and that the content is accessible to screen readers. Accessible means that it is not hidden using the aria-hidden prop. Refer to the references to learn about why this is important.


## Bad Practice

```javascript
<h1 />
<h1><TextWrapper aria-hidden />
```

## Recommended

```javascript
<h1>Heading Content!</h1>
<h1><TextWrapper /><h1>
<h1 dangerouslySetInnerHTML={{ __html: 'foo' }} />
```
