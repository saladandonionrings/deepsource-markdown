# Detected generic Array constructors
**ID:** `JS-0316` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0316)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Use of the `Array` constructor to construct a new array is generally discouraged in favor of array literal notation.

The `Array` global may be redefined, and thus could refer to something other than the actual `Array` class. It is wiser to use array literals when favorable. The exception is when the Array constructor is used to intentionally create sparse arrays of specified sizes by giving the constructor a single numeric argument.


## Bad Practice

```javascript
Array(0, 1, 2);
new Array(0, 1, 2);
```

## Recommended

```javascript
[0, 1, 2]

Array(500);
new Array(someOtherArray.length);
```
