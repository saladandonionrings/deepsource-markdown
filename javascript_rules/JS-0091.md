# Throwing literals as exceptions is not recommended
**ID:** `JS-0091` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0091)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

It is considered good practice to only `throw` the `Error` object itself or an object using the `Error` object as base objects for user-defined exceptions. The benefit of `Error` objects is that they automatically keep track of where they were built and originated. This rule restricts what can be thrown as an exception.


## Bad Practice

```javascript
throw "error";

throw 0;

throw undefined;

throw null;

var err = new Error();
throw "an " + err;
// err is recast to a string literal

var err = new Error();
throw `${err}`
```

## Recommended

```javascript
throw new Error();

throw new Error("error");

var e = new Error("error");
throw e;

try {
    throw new Error("error");
} catch (e) {
    throw e;
}
```
