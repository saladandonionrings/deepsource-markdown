# Use `const` declarations for variables that are never reassigned
**ID:** `JS-0242` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0242)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Variables that are never re-assigned a new value after their initial declaration should be declared with the `const` keyword. This prevents the programmer from erroneously re-assigning to a read-only variable, and informs those reading the code that a variable is a constant value.


## Bad Practice

```javascript
let pi = Math.PI

for (let x of xs) {
  use(x);
}

let { a, b } = object;
use(a, b);
```

## Recommended

```javascript
const pi = Math.PI

for (const x of xs) {
  use(x);
}

const { a, b } = object;
use(a, b);
```
