# Detected the `delete` operator with computed key expressions
**ID:** `JS-0320` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0320)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Deleting dynamically computed keys can be dangerous and in some cases not well optimized.

Using the `delete` operator on keys that aren't runtime constants could be a sign that you're using the wrong data structures. Using `Objects` with added and removed keys can cause occasional edge case bugs, such as if a key is named `"hasOwnProperty"`. Consider using a `Map` or `Set` if you’re storing collections of objects.


## Bad Practice

```javascript
// Can be replaced with the constant equivalents, such as container.aaa
delete container['aaa'];
delete container['Infinity'];

// Dynamic, difficult-to-reason-about lookups
const name = 'name';
delete container[name];
delete container[name.toUpperCase()];
```

## Recommended

```javascript
const container: { [i: string]: number } = {
  /* ... */
};

// Constant runtime lookups by string index
delete container.aaa;

// Constants that must be accessed by []
delete container[7];
delete container['-Infinity'];
```
