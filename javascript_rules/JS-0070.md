# The use of the `__iterator__` property is not preferred
**ID:** `JS-0070` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0070)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

The `__iterator__` property was a SpiderMonkey extension to JavaScript that could be used to create custom iterators that are compatible with JavaScript's `for in` and `for each` constructs. However, this property is now obsolete, so it should not be used. Here's an example of how this used to work:


```javascript
Foo.prototype.__iterator__ = function() {
    return new FooIterator(this);
}
```
Now, you should use ECMAScript 6 iterators and generators instead.


## Bad Practice

```javascript
Foo.prototype.__iterator__ = function() {
    return new FooIterator(this);
};

foo.__iterator__ = function () {};

foo["__iterator__"] = function () {};
```

## Recommended

```javascript
var __iterator__ = foo; // Not using the `__iterator__` property.
```
