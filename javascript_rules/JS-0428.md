# Avoid using spreading operator for JSX props
**ID:** `JS-0428` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0428)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

We recommend passing props separately to the component instead of using a spread operator for any JSX props.

Here are some reasons to avoid the use of the spreading operator:

* Readability -- being more explicit about what props are received by the component is straightforward.* Maintainability -- helps avoid passing unintentional extra props.* `React`emit warnings when invalid HTML props are passed to HTML elements. This is suppressed when the spreading operator is used.

## Bad Practice

```javascript
<App {...props} />
<MyCustomComponent {...props} some_other_prop={some_other_prop} />
<img {...props} />
```

## Recommended

```javascript
const {src, alt} = props;
const {one_prop, two_prop} = otherProps;
<MyCustomComponent one_prop={one_prop} two_prop={two_prop} />
<img src={src} alt={alt} />
```
