# Avoid insecure dns prefetch control configuration for helmet
**ID:** `JS-S1003` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-S1003)

![Critical](https://img.shields.io/badge/severity-critical-red)![Security](https://img.shields.io/badge/type-security-red)

Whenever the user clicks on a link in a webpage, the link must be resolved to an IP address before the user can be served. DNS prefetching is a feature that allows browsers to perform DNS resolution in the background, even before the user has clicked on a link. This helps reduce load times for the target web-page. However, it is recommended to set this header to `off`, as it can bear significant performance costs when the webpage is in an idle state.

Moreover, DNS-prefetching bears **privacy issues**. The user's behavioral data is exposed to the server even before the user has visited a webpage, and DNS-prefetching is performed without his consent.

In NodeJS, these headers can be controlled using a popular third-party library named `helmet`.


## Bad Practice

```javascript
import express from 'express'
import helmet from 'helmet'

const app = express()
app.use(
  helmet.dnsPrefetchControl({
    allow: true // Has privacy and performance implications 
  })
)
```

## Recommended

```javascript
import express from 'express'
import helmet from 'helmet'

const app = express()
app.use(
  helmet.dnsPrefetchControl({
    allow: false // Safe
  })
)
```
