# Use shorthand promise methods
**ID:** `JS-C1004` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-C1004)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

If the executor argument to a promise constructor only calls the `reject` or `resolve` and exits, then the code can refactored to use `Promise#reject` or `Promise#resolve` instead.


## Bad Practice

```javascript
new Promise((resolve, reject) => {
  resolve(getItem())
})

new Promise(function (resolve, reject) {
  reject("oops")
})
```

## Recommended

```javascript
Promise.resolve(getItem())
Promise.reject("oops")
```
