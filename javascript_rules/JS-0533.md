# Prefer having a prefix for all component names
**ID:** `JS-0533` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0533)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

It is recommended to use consistent nomenclature of components across the codebase. Consistency within the project is important for the team because it provides tremendous efficiency. The naming conventions should help you find your code faster and make it easier to understand. We recommend not to prefix the components by `ng` as it is a reserved keyword for AngularJS components. This issue is recommended for Angular v1.x versions.

Note: This issue is reported when the project has an existing ESLint configuration file.


## Bad Practice

```javascript
// error: The navigation component should follow this pattern: /^ui/
angular
    .module('myModule')
    .component('navigation', {
        // ...
    });
```

## Recommended

```javascript
angular
    .module('myModule')
    .component('prefixTabs', {
        // ...
    });
```
