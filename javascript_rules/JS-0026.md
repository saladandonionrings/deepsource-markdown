# Avoid using control flow statements in `finally` blocks
**ID:** `JS-0026` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0026)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

The usage of control flow statements like `return` , `throw` , `break` , and `continue` inside `finally` blocks is not recommended. JavaScript will suspend the execution of any control flow statements in the `try` and/or `catch` blocks and will run the `finally` block first. For example, a return within a `finally` block would override a return within the preceding `try` block, often leading to unexpected behavior.


## Bad Practice

```javascript
// We expect this function to return 1;
const foo = function() {
    try {
        return 1; // 1 is returned but suspended until finally block ends
    } catch(err) {
        return 2;
    } finally {
        throw new Error(); // Error was thrown before returning 1
    }
}
```

## Recommended

```javascript
let foo = function() {
    try {
        return 1;
    } catch(err) {
        return 2;
    } finally {
        console.log("hola!");
    }
};
```
