# Use `RegExp#exec` instead of `String#match`
**ID:** `JS-0370` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0370)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

`RegExp#exec` is faster than `String#match` and both work the same when not using the `/g` flag.


## Bad Practice

```javascript
'something'.match(/thing/);

'some things are just things'.match(/thing/);

const text = 'something';
const search = /thing/;
text.match(search);
```

## Recommended

```javascript
/thing/.exec('something');

'some things are just things'.match(/thing/g);

const text = 'something';
const search = /thing/;
search.exec(text);
```
