# Missing target origin in cross-origin communication
**ID:** `JS-S1014` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-S1014)

![Critical](https://img.shields.io/badge/severity-critical-red)![Security](https://img.shields.io/badge/type-security-red)

The [window.postMessage](https://developer.mozilla.org/en-US/docs/Web/API/Window/postMessage) function is used for cross-origin communication between two entities. An "entity" here can be a page, an iframe, or a pop-up spawned by a page. It is recommended to always have an explicit [targetOrigin parameter](https://developer.mozilla.org/en-US/docs/Web/API/Window/postMessage#parameters) when using `window.postMessage`. This allows the message to be sent only to trusted receivers - those that have the required URI.

When sharing a password for example, it is critical that the password only be sent to the intended receiver, and not some malicious third party.


```javascript
const password = getPassword()
window.postMessage(password, "https://safe.site.com")
```
If you have no preference for the origin of a window for a message to be dispatched, you can use `*`:


```javascript
window.postMessage(data, "*")
```

## Bad Practice

```javascript
// send a message from a pop-up:
const popup = window.open(/* popup data */)

// INSECURE: The message can be received by any window that is currently active
// below the pop-up.
popup.postMessage("secret message for the parent")
```

## Recommended

```javascript
// send a message from a pop-up:
const popup = window.open(/* popup data */)

// SAFE: The message can only be received by example.com 
popup.postMessage("secret message for the parent", "https://www.example.com")
```
