# Avoid using the return value of `ReactDOM.render`
**ID:** `JS-0449` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0449)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

`ReactDOM.render()` currently returns a reference to the root `ReactComponent` instance. However, using this return value is legacy and should be avoided because future versions of React may render components asynchronously in some cases.

If you need a reference to the root ReactComponent instance, the preferred solution is to attach a callback ref to the root element.


## Bad Practice

```javascript
const inst = ReactDOM.render(<App />, document.body);
doSomethingWithInst(inst)
```

## Recommended

```javascript
ReactDOM.render(<App ref={doSomethingWithInst} />, document.body);

ReactDOM.render(<App />, document.body, doSomethingWithInst);
```
