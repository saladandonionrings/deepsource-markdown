# Require `$on` and `$watch` deregistration callbacks to be saved in a variable
**ID:** `JS-0529` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0529)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Watch and On methods on the scope object should be assigned to a variable, in order to be deleted in a `$destroy` event handler **Rule based on Angular 1.x**


## Bad Practice

```javascript
$rootScope.$on('event', function () {
    // ...
});
```

## Recommended

```javascript
$scope.$on('event', function () {
    // ...
});

var unregister = $rootScope.$on('event', function () {
    // ...
});
```
