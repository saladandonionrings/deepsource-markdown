# Found empty glimmer component classes
**ID:** `JS-P1001` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-P1001)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Performance](https://img.shields.io/badge/type-performance-white)

The issue is raised when it founds use of empty backing classes for Glimmer components because Template-only Glimmer components with no backing class is much faster than Glimmer components with backing classes due to lighter-weight than Ember Components. Therefore, you should only have a backing class for a Glimmer component when absolutely necessary.

To fix violations of this issue:

* In apps: Remove the backing class entirely until it is actually needed.* In in-repo addons: Replace the backing class depending on what the host app is doing. That is, if`template-only-glimmer-components`is enabled, remove the backing class. Otherwise, replace it with a`templateOnly`export.* In other addons: Replace the backing class with a`templateOnly`export. This is necessary because you can't assume`template-only-glimmer-components`is enabled.

## Bad Practice

```javascript
import Component from '@glimmer/component';

class MyComponent extends Component {}
```

## Recommended

```javascript
import Component from '@glimmer/component';

class MyComponent extends Component {
  glimmerfunction() {
    return this.args.arg_one + this.args.arg_two;
  }
}
```

```javascript
import templateOnly from '@ember/component/template-only';

const MyComponent = templateOnly();

export default MyComponent;
```
