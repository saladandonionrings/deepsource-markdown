# Detected unnecessary equality comparisons against boolean literals
**ID:** `JS-0345` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0345)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Comparing boolean values to boolean literals is unnecessary, those comparisons result in the same booleans. Using the boolean values directly, or via a unary negation (`!value`), is more concise and clearer.


## Bad Practice

```javascript
declare const someCondition: boolean;
if (someCondition === true) {
}
```

## Recommended

```javascript
declare const someCondition: boolean;
if (someCondition) {
}

declare const someObjectBoolean: boolean | Record<string, unknown>;
if (someObjectBoolean === true) {
}

declare const someStringBoolean: boolean | string;
if (someStringBoolean === true) {
}

declare const someUndefinedCondition: boolean | undefined;
if (someUndefinedCondition === false) {
}
```
