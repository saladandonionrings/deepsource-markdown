# Found division operators explicitly at the beginning of regular expressions
**ID:** `JS-0055` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0055)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Regex literals should escape division operators.


## Bad Practice

```javascript
function bar() { return /=foo/; }
```

## Recommended

```javascript
function bar() { return /[=]foo/; }
```
