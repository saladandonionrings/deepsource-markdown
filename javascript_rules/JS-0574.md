# Invalid naming of directive outputs
**ID:** `JS-0574` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0574)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

We should not use output and input names that collide with either a native property/event name or even a name used by another component or directive. A name should clearly convey the meaning of the event because listeners subscribed to an output with such a name will also be invoked when the native event is raised.


## Bad Practice

```javascript
@Component()
class Test {
    @Output() change = new EventEmitter<string>();
}
```

```javascript
@Directive()
class Test {
    @Output() change = new EventEmitter<string>();
}
```

## Recommended

```javascript
@Component()
class Test {
    @Output() buttonChange = new EventEmitter<string>();
}
```

```javascript
@Directive()
class Test {
    @Output('buttonChange') _buttonChange = new EventEmitter<string>();
}
```
