# Found shorthand type coercions
**ID:** `JS-0066` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0066)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Prefer using explicit casts by calling `Number` , `Boolean` , or `String` over using operators like `+` , `!!` or `"" +` . This is considered best practice as it improves readability.


## Bad Practice

```javascript
const b = !!foo;
// The `+` operator does not change the value of its operand
// unless it's already a number.
let n = +foo;
n = 1 * foo;
const s = "" + foo;
```

## Recommended

```javascript
const b = Boolean(foo);
const n = Number(foo);
const s = String(foo);
```
