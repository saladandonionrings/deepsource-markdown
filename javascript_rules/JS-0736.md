# Should not use template `parentData()`
**ID:** `JS-0736` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0736)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

When making children aware of their parents data context, they are tightly integrated and hard to reuse. Changing the parent can lead to unintended errors in the child. Passing down the properties explicitly avoids this issue.


## Bad Practice

```javascript
Template.parentData()
Template.parentData(0)
Template.parentData(1)
Template.parentData(foo)
```

## Recommended

```javascript
Template.currentData()
```
