# Prefer using return type annotations for functions
**ID:** `JS-0499` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0499)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Return types ensure that every branch of your function returns the same type. This prevents us from accidentally not returning a value under certain conditions or returning an incorrect type.

These return types also help in determining the type of the variable during an assignment expression. for example


```javascript
function concatenateNumberAndStringFn(a: number, b: string) {
    return () => a + b
}

const fn = concatenateNumberAndStringFn(1, "val")
```
A developer might confuse whether the `concatenateNumberAndStringFn` is returning from the addition or a function or anything else. Having a return type will solve this issue.


```javascript
function concatenateNumberAndStringFn(a: number, b: string): () => string {
    return () => a + b
}

const fn = concatenateNumberAndStringFn(1, "val")
```
Here we know that `fn` is of type `() => string` . That is, it is a function rather than a value. There are many other similar benefits of having proper return type and improving type checking and developer experience.


## Bad Practice

```javascript
(data) => { return "data"; }

(data) => { return "data"; }

(data) => "data"

(data) => ({})

(data): undefined => { return; }

(data): void => { return; }

(data): undefined => { return undefined; }

(data): void => { return void 0; }
```

## Recommended

```javascript
return;

(data): string => {}

const f: Foo = (a, b) => 42;

(data): string => {}

type fn = (a: string, b: number) => number;
const f: fn = (a, b) => { return 42; }

(data) => { return; }

(data): Object => ( {} )

(data) => { return undefined; }

(data) => { return void 0; }
```
