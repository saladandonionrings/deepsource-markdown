# Usage of octal escape sequences in string literals is not preferred
**ID:** `JS-0082` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0082)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

As of the ECMAScript 5 specification, octal escape sequences in string literals are deprecated and should not be used. Unicode escape sequences should be used instead.


```javascript
const foo = "Copyright \251";
```

## Bad Practice

```javascript
const foo = "Copyright \251";
```

## Recommended

```javascript
let foo = "Copyright \u00A9";   // unicode
foo = "Copyright \xA9";     // hexadecimal
```
