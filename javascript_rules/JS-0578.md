# Invalid async pipe
**ID:** `JS-0578` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0578)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Angular’s async pipes emit null initially, prior to the observable emitting any values, or the promise resolving.

This can cause negations, like `*ngIf=”!(myConditional | async)”` to thrash the layout and cause expensive side-effects like firing off XHR requests for a component which should not be shown.


## Bad Practice

```javascript
{{ (foo | async) == false }}
{{ !(foo | somethingElse | async) }}
<div *ngIf="!(a | async)"></div>
```

## Recommended

```javascript
{{ (foo | async) }}
{{ !(foo | async | somethingElse) }}
{{ !(foo | notAnAsyncPipe) }}
{{ (foo | async) === false }}
```
