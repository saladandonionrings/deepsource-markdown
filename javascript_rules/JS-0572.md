# `@Attribute` decorator should not be used
**ID:** `JS-0572` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0572)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

We can specify the attribute name as a parameter of the `@Attribute` decorator. It is a good practice to specify this attribute name directly by naming the corresponding parameter, in order to avoid duplicate code.


## Bad Practice

```javascript
class Test {
   constructor(@Attribute() foo: any) {}
}
```

```javascript
class Test {
   constructor(@Attribute() foo: any, @Attribute() bar: any) {}
}
```

## Recommended

```javascript
class Test {
   constructor(@Optional() foo: any, @Optional() bar: any) {}
}
```

```javascript
class Test {
   constructor(foo: any, @Optional() bar: any) {}
}
```
