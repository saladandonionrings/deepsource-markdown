# Found constant expressions in conditions
**ID:** `JS-0003` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0003)

![Major](https://img.shields.io/badge/severity-major-orange)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

A constant expression such as a literal when used as a condition in an `if`/ `for` / `while` or `do...while` statement can cause errors in a production environment.

Usage of such constructs in a development environment for debugging triggers is common, but it's not a good practice to push them to VCS.


## Bad Practice

```javascript
if (false) {
    doSomethingUnfinished();
}

if (void x) {
    doSomethingUnfinished();
}

for (;-2;) {
    doSomethingForever();
}

while (typeof x) {
    doSomethingForever();
}

do {
    doSomethingForever();
} while (x = -1);

const result = 0 ? a : b;
```

## Recommended

```javascript
if (foo()) {
    doSomethingUnfinished();
}

for (let i = 0; i < foo; ++i) {
    doSomethingForever();
}

while (true) {
    doSomethingForever();
}

do {
    doSomethingForever();
} while (x == -1);

const result = cond() ? a : b;
```
