# `Ember.testing` not allowed in modules scope
**ID:** `JS-0783` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0783)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

`Ember.testing` is not allowed in modules scope. Since `[email protected]` / `[email protected]` , `Ember.testing` is only set to `true` while a test is executing instead of all the time. Also, `Ember.testing` is a getter/setter in Ember and destructuring will only read its value at the time of destructuring.


## Bad Practice

```javascript
import Component from '@ember/component';

export default Component.extend({
  init(...args) {
    this._super(...args);
    this.isTesting = Ember.testing;
  }
});
```

```javascript
import Ember from 'ember';
import Component from '@ember/component';

export default Component.extend({
  isTesting: Ember.testing
});
```

```javascript
import Ember from 'ember';

const IS_TESTING = Ember.testing;
```

```javascript
import Ember from 'ember';

const { testing } = Ember;
```

## Recommended

```javascript
import Ember from 'ember';
import Component from '@ember/component';

export default Component.extend({
  someMethod() {
    if (Ember.testing) {
      doSomething();
    } else {
      doSomethingElse();
    }
  }
});
```

```javascript
import Ember from 'ember';
import Service from '@ember/service';

export default Service.extend({
  foo() {
    _bar(Ember.testing ? 0 : 400);
  }
});
```
