# Avoid usage of deprecated `$listeners` on `v-on`
**ID:** `JS-0644` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0644)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

`$listeners` is used for creating transparent wrapper components because it contains parent-scope `v-on` event listeners wihtout `.native` modifiers. After the release of Vue3, `this.$attrs` now contains everything passed to the component minus those explicitly declared as `props` , including `class` , `style` , and `v-on` listeners. `this.$listeners` is removed.


## Bad Practice

```javascript
<template>
  <label>
    <input type="text" v-bind="$attrs" v-on="$listeners" />
  </label>
</template>

<script>
  export default {
    inheritAttrs: false
  }
</script>
```

## Recommended
In Vue 3's virtual DOM, event listeners are now just attributes, prefixed with on, and as such are part of the $attrs object, so $listeners has been removed.


```javascript
<template>
  <label>
    <input type="text" v-bind="$attrs" />
  </label>
</template>

<script>
export default {
  inheritAttrs: false
}
</script>
```
