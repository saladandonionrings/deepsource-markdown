# Don't use Ember's `function` prototype
**ID:** `JS-0784` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0784)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Do not use Ember's `function` prototype extensions.

Use computed property syntax, observer syntax or module hooks instead of `.property()` , `.observes()` or `.on()` in Ember modules.


## Bad Practice

```javascript
export default Component.extend({
  abc: function () {
    /* custom logic */
  }.property('xyz'),
  def: function () {
    /* custom logic */
  }.observes('xyz'),
  ghi: function () {
    /* custom logic */
  }.on('didInsertElement')
});
```

## Recommended

```javascript
export default Component.extend({
  abc: computed('xyz', function () {
    /* custom logic */
  }),
  def: observer('xyz', function () {
    /* custom logic */
  }),
  didInsertElement() {
    /* custom logic */
  }
});
```
