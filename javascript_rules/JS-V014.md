# Invalid calls after `await`
**ID:** `JS-V014` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-V014)

![Major](https://img.shields.io/badge/severity-major-orange)![Performance](https://img.shields.io/badge/type-performance-white)

Vue lifecycle hook registration functions can only be used synchronously during `setup()`, since they rely on internal global state to locate the current active instance (the component instance whose `setup()` is being called right now). Calling them without a current active instance will result in an error.

Asynchronous lifecycle hooks will slow down render and may delay rendering of other components of the app that don’t have to wait for the loading to finish. This will have undesired effects like making lifecycle hooks rendering blocking affecting performance and UX.


## Bad Practice

```javascript
<script>
import { useI18n } from 'vue-i18n'
export default {
  async setup() {

    await doSomething()

    /* ✗ BAD */
    useI18n({})
  }
}
</script>
```

## Recommended

```javascript
<script>
import { useI18n } from 'vue-i18n'
export default {
  async setup() {
    /* ✓ GOOD */
    useI18n({})

    await doSomething()
  }
}
</script>
```
