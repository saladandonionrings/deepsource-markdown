# Deprecation of `destroyed` and `beforeDestroy` lifecycle hooks
**ID:** `JS-0647` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0647)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Following the release of Vue 3.0, the following changes have been made:

* Filters also create extra complexity in template IDE support (again due to them not being really JavaScript).
* In fact, it conflicts with JavaScript's own bitwise or operator ( `|` ) and makes expression parsing more complicated.


## Bad Practice

```javascript
{{ msg | filter }}
  {{ msg | filter('€') }}
  {{ msg | filterA | filterB }}
  <div v-bind:id="msg | filter"></div>
  <div v-bind:id="msg | filter('€')"></div>
  <div v-bind:id="msg | filterA | filterB"></div>
```

## Recommended

```javascript
{{ filter(msg) }}
  {{ filter(msg, '€') }}
  {{ filterB(filterA(msg)) }}
  <div v-bind:id="filter(msg)"></div>
  <div v-bind:id="filter(msg, '€')"></div>
  <div v-bind:id="filterB(filterA(msg))"></div>
```
