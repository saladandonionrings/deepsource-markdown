# Ensure `style` attribute is an `Object`
**ID:** `JS-0473` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0473)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

This issue is raised when the `style` attribute has a non-object value in the component.

The issue can be fixed by assigning an object `{}` to the `style` attribute.


## Bad Practice

```javascript
// Example 1
<div style="color: 'red'" />
<div style={true} />
<Hello style={true} />

// Example 2
React.createElement("div", { style: "color: 'red'" });
React.createElement("div", { style: true });

// Example 3
const styles = true;
React.createElement("div", { style: styles });
<div style={styles} />
```

## Recommended

```javascript
// Example 1
<div style={{ color: "red" }} />
<Hello style={{ color: "red" }} />
const styles = { color: "red" };

// Example 2
React.createElement("div", { style: { color: 'red' }});
React.createElement("Hello", { style: { color: 'red' }});

// Example 3
const styles = { height: '100px' };
React.createElement("div", { style: styles });
<div style={styles} />
```
