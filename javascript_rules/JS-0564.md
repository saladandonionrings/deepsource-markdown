# Prefer to use `angular.isDate` instead of `typeof` comparisons
**ID:** `JS-0564` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0564)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

You should use the angular.isDate method instead of the default JavaScript implementation (typeof new Date() === "object Date").

**Rule based on Angular 1.x**


## Bad Practice

```javascript
// invalid
Object.prototype.toString.call(someDate) === '[object Date]'; // error: You should use the angular.isDate method
```

## Recommended

```javascript
// valid
angular.isDate(someDate);
```
