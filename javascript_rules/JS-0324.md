# Unnecessary non-null assertion
**ID:** `JS-0324` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0324)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Multiple non-null assertion operators (`!`s) are redundant, and may confuse the reader. Moreover, `foo?.bar` should always be preferred over `foo!?.bar`.


## Bad Practice

```javascript
const foo: { bar: number } | null = null;
const bar = foo!!!.bar;

function foo(bar: number | undefined) {
  const bar: number = bar!!!;
}

function foo(bar?: { n: number }) {
  return bar!?.n;
}
```

## Recommended

```javascript
const foo: { bar: number } | null = null;
const bar = foo!.bar;

function foo(bar: number | undefined) {
  const bar: number = bar!;
}

function foo(bar?: { n: number }) {
  return bar?.n;
}
```
