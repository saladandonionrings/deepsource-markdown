# Found assignment operators in conditional expressions
**ID:** `JS-0001` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0001)

![Major](https://img.shields.io/badge/severity-major-orange)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

In conditional statements, it is very easy to mistype a comparison operator (such as `==` or `===`) as an assignment operator (such as `=`).

## Bad Practice

```javascript
// Unintentional assignment
let x;
if (x = 0) {
    let b = 1;
}
```

## Recommended

```javascript
// Assignment replaced by comparison
let x;
if (x === 0) {
    let b = 1;
}
let y;
if (y === 0) {
    let b = 1;
}
```
