# Detected the use of custom TypeScript modules and namespaces
**ID:** `JS-0337` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0337)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Custom TypeScript modules (`module foo {}`) and namespaces (`namespace foo {}`) are considered outdated ways to organize TypeScript code. ES2015 module syntax is now preferred (`import/export`).

This rule still allows the use of TypeScript module declarations to describe external APIs (`declare module 'foo' {}`).


## Bad Practice

```javascript
module foo {}
namespace foo {}

declare module foo {}
declare namespace foo {}
```

## Recommended

```javascript
declare module 'foo' {}
```
