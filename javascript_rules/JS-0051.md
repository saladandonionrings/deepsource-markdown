# Prefer that `for-in` loops should include an `if` statement
**ID:** `JS-0051` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0051)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Looping over objects with a `for in` loop will include properties that are inherited through the prototype chain. This behavior can lead to unexpected keys in your for loop.


## Bad Practice

```javascript
for (const key in foo) {
    doSomething(key);
}
```

## Recommended

```javascript
for (const key in foo) {
    if (Object.prototype.hasOwnProperty.call(foo, key)) {
        doSomething(key);
    }
}

for (const key of Object.keys(foo)) {
    doSomething(key);
}
```
