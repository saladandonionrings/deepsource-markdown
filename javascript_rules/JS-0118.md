# Usage of `strict` mode against recommended approach
**ID:** `JS-0118` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0118)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

A strict mode directive is a `"use strict"` literal at the beginning of a script or function body. It enables strict mode semantics.

When a directive occurs in global scope, strict mode applies to the entire script:


```javascript
"use strict";

// strict mode

function foo() {
    // strict mode
}
```
When a directive occurs at the beginning of a function body, strict mode applies only to that function, including all contained functions:


```javascript
function foo() {
    "use strict";
    // strict mode
}

function foo2() {
    // not strict mode
};

(function() {
    "use strict";
    function bar() {
        // strict mode
    }
}());
```
In the CommonJS module system, a hidden function wraps each module and limits the scope of a "global" strict mode directive.

In ECMAScript modules, which always have strict mode semantics, the directives are unnecessary.


## Bad Practice

```javascript
function foo() {
}

function foo() {
    "use strict";
}

"use strict";

function foo() {
    "use strict";
}
```

## Recommended

```javascript
function foo() {
}
```
