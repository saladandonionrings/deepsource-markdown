# Detected a namespace qualifier is unnecessary
**ID:** `JS-0347` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0347)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

We need to know when a namespace or enum qualifier is unnecessary, whether used for a type or for a value.


## Bad Practice

```javascript
// Example 1
namespace A {
  export type B = number;
  const x: A.B = 3;
}

// Example 2
namespace A {
  export const x = 3;
  export const y = A.x;
}

// Example 3
enum A {
  B,
  C = A.B,
}

// Example 4
namespace A {
  export namespace B {
    export type T = number;
    const x: A.B.T = 3;
  }
}
```

## Recommended

```javascript
// Example 1
namespace X {
  export type T = number;
}

namespace Y {
  export const x: X.T = 3;
}

// Example 2
enum A {
  X,
  Y,
}

enum B {
  Z = A.X,
}

// Example 3
namespace X {
  export type T = number;
  namespace Y {
    type T = string;
    const x: X.T = 0;
  }
}
```
