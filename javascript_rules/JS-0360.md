# Prefer usage of `as const` over literal type
**ID:** `JS-0360` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0360)

![Major](https://img.shields.io/badge/severity-major-orange) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

It recommends usage of `as const` assertion when type primitive value is equal to type.


## Bad Practice

```javascript
const two: 2 = 2 as 2;
const bar = <'bar'>'bar';
```

## Recommended

```javascript
const foo = 'foo';
const bar: 'bar' = 'bar' as const;
```
