# Found deprecated `*` existential type
**ID:** `JS-0484` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0484)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

The use of the `*` (existential) type is not recommended, as this type is unsafe and usually just equivalent to `any` . The same effect can generally be achieved by simply not providing a type annotation.


## Bad Practice

```javascript
type T = *;
type T = U<*, *>;
const f: (*) => null = () => null;
```

## Recommended

```javascript
type T = string | null
```
