# Getter without a setter pair in objects
**ID:** `JS-0041` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0041)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

It's a common mistake in JavaScript to create an object with just a setter for a property but never have a corresponding getter defined for it. Without a getter, you cannot read the property, so it ends up not being used.


## Bad Practice

```javascript
const o = {
    set a(value) {
        this.val = value;
    }
};
// or
const o = {d: 1};
Object.defineProperty(o, 'c', {
    set: function(value) {
        this.val = value;
    }
});
```

## Recommended

```javascript
const o = {
    set a(value) {
        this.val = value;
    },
    get a() {
        return this.val;
    }
};

const o = {d: 1};
Object.defineProperty(o, 'c', {
    set: function(value) {
        this.val = value;
    },
    get: function() {
        return this.val;
    }
});
```
