# Multiple `<Head>` detected in `pages/document.js`
**ID:** `JS-W1023` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-W1023)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

It is not recommended to use more than a single instance of `Head` component in a single custom document. This can cause unexpected behavior in your application.


## Bad Practice

```javascript
<!-- pages/document.js -->
import Document, { Html, Main, NextScript } from "next/document";
import Head from "next/head";

class MyDocument extends Document {
  render() {
    return (
      <Html>
        <Head />
        <WrapperNavbar>
          <Head />
        </WrapperNavbar>
      </Html>
    );
  }
}

export default MyDocument;
```

## Recommended

```javascript
<!-- pages/document.js -->
import Document, { Html, Main, NextScript } from "next/document";
import Head from "next/head";

class MyDocument extends Document {
  render() {
    return (
      <Html>
        <Head />
      </Html>
    );
  }
}

export default MyDocument;
```
