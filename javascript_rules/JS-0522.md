# Found unused dependency injection parameters
**ID:** `JS-0522` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0522)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Dependency Injection (DI) is a software design pattern that deals with how components get hold of their dependencies. The AngularJS injector subsystem is in charge of creating components, resolving their dependencies, and providing them to other components as requested. It is used when defining components or when providing `run` and `config` blocks for a module.

Consider removing unused dependency injection function parameters.


## Bad Practice

```javascript
angular
    .module('myModule')
    .factory('myService', function ($http, $q, $log) {
        $http.get('/api/someData').then(function (response) {
            $log.log(response.data);
        });
    });
```

## Recommended

```javascript
angular
    .module('myModule')
    .factory('myService', function ($http, $log) {
        $http.get('/api/someData').then(function (response) {
            $log.log(response.data);
        });
    });
```
