# Definitions for unused states detected
**ID:** `JS-0458` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0458)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Avoid defining states that are not being used. Unused state serves no purpose and is essentially dead code.

* This happens commonly when there is a lack of typings or during extensive migrations or making the codebase more reusable.* This also occurs when developers write big components. Like a single component that does more than one task.* It is better to have a single component for one job. This way, the data flow within and across components is more apparent.

## Bad Practice

```javascript
class MyComponent extends React.Component {
  state = { foo: 0 };
  render() {
    return <SomeComponent />;
  }
}

const UnusedGetInitialStateTest = createReactClass({
  getInitialState: function() {
    return { foo: 0 };
  },
  render: function() {
    return <SomeComponent />;
  }
})
```

## Recommended

```javascript
class MyComponent extends React.Component {
  state = { foo: 0 };
  render() {
    return <SomeComponent foo={this.state.foo} />;
  }
}

const UnusedGetInitialStateTest = createReactClass({
  getInitialState: function() {
    return { foo: 0 };
  },
  render: function() {
    return <SomeComponent foo={this.state.foo} />;
  }
})
```
