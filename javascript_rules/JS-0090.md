# Usage of comma operators should be avoided
**ID:** `JS-0090` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0090)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

The comma operator includes multiple expressions where only one is expected. It evaluates each operand from left to right and returns the value of the last operand. However, this frequently obscures side effects, and its use is often an accident. Here are some examples of sequences:


## Bad Practice

```javascript
let a = (3, 5); // a = 5
a = b += 5, a + b;
while (a = next(), a && a.length);
(0, eval)("doSomething();");
res = doSomething(), val;
0, eval("doSomething();");
do {} while (doSomething(), !!test);
for (; doSomething(), !!test; );
if (doSomething(), !!test);
switch (val = func(), val) {}
while (val = func(), val < 42);
with (doSomething(), val) {}
```

## Recommended

```javascript
res = (doSomething(), val);
(0, eval)("doSomething();");
do {} while ((doSomething(), !!test));
for (i = 0, j = 10; i < j; i++, j--);
if ((doSomething(), !!test));
switch ((val = func(), val)) {}
while ((val = func(), val < 42));
with ((doSomething(), val)) {}
```
