# Use `isNaN()` to check for `NaN`
**ID:** `JS-0027` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0027)

![Major](https://img.shields.io/badge/severity-major-orange)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Consider using the `isNaN` function instead of comparing the value with `NaN`. `NaN === NaN` always evaluates to `false` .

In JavaScript, `NaN` is a unique value of type `Number`. It's used to represent any of the "not-a-number" values represented by the double-precision 64-bit format specified by the [IEEE Standard for Binary Floating-Point Arithmetic](https://en.wikipedia.org/wiki/IEEE_754).

Because `NaN` is unique in JavaScript by not being equal to anything, including itself, the results of comparisons to NaN are confusing:

* `NaN === NaN`and`NaN == NaN`evaluate to`false`* `NaN !== NaN`and`NaN != NaN`evaluate to`true`
Therefore, it's better to use `Number.isNaN()` or the global `isNaN()` function to test whether a value is `NaN`.


## Bad Practice

```javascript
if (TTY == NaN) {
    // ...
}

if (TTY != NaN) {
    // ...
}
```

## Recommended

```javascript
if (isNaN(TTY)) {
    // ...
}

if (!isNaN(TTY)) {
    // ...
}
```
