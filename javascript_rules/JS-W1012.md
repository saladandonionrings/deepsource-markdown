# Avoid using `setTimeout/setInterval` in `asyncData/fetch`
**ID:** `JS-W1012` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-W1012)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

It is recommended not to use `setTimeout` and `setInterval` in `asyncData` and `fetch` hooks. `asyncData` returns data directly and merges it into the component (page) `data` which might lead to a memory leak.

`asyncData` is called every time before loading the page component and is only available for such. It will be called on server-side once (on the first request to the Nuxt app) and client-side when navigating to further routes. This method receives the `context` object as the first argument; you can use it to fetch some data and return the component data. The `fetch` hook (from `Nuxt 2.12+` ) can be placed on any component and provides shortcuts for rendering loading states (during client-side rendering) and errors.


## Bad Practice

```javascript
export default {
  asyncData() {
    let name = 'joe'
    setTimeout(() => {
      name = 'baz'
    }, 0)
  },
  fetch() {
    let name = 'joe'
    setInterval(() => {
      name = 'smith'
    }, 0)
  }
}
```

## Recommended

```javascript
export default {
  async asyncData() {
    let name = 'smith'
  },
  fetch() {
    let name = 'smith'
  }
}
```
