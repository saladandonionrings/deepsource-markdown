# Avoid using `process.server`, `process.client` and `process.browser` in client side hooks
**ID:** `JS-E1000` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-E1000)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

It is recommended to avoid using `process.server` , `process.client` and `process.browser` in the following hooks :

* beforeMount
* mounted
* beforeUpdate
* updated
* activated
* deactivated
* beforeDestroy
* destroyed

The aforementioned hooks are available on the client-side. The `process.server` , `process.client` and `process.browser` are available on the server-side, and accessing them in client side hooks might result in unexpected errors or even result as `undefined` .


## Bad Practice

```javascript
export default {
  mounted() {
    if (process.server) {
      const city = 'NYC'
    }
  },
  beforeMount() {
    if (process.client) {
      const city = 'NYC'
    }
  },
  beforeDestroy() {
    if (process.browser) {
      const city = 'NYC'
    }
  }
}
```

## Recommended

```javascript
export default {
  mounted() {
    const city = 'NYC'
  },
  beforeMount() {
    const city = 'NYC'
  },
  beforeDestroy() {
    const city = 'NYC'
  }
}
```
