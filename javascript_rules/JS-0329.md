# Found `for-in` loop iterating over an array
**ID:** `JS-0329` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0329)

![Major](https://img.shields.io/badge/severity-major-orange)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

A `for...in` loop (`for (var k in o)`) iterates over an `Object` properties. While it is legal to use `for...in` loops with array types, it is not common. `for...in` will iterate over the indices of the array as strings, omitting any holes in the array. More common is to use `for...of`, which iterates over the values of an array.

While both `for-in` and `for-of` is used for looping , there is one difference

* `for-in`iterates over enumerable properties of an object. They contains the`[[Enumerable]]`attribute* `for-of`iterates over values of the iteratable objects. example arrays.
So looping in array should be done with `for-of` as they dont contain enumerable properties.

Another reason to prefer `for-of` over `for-in` is that `for-in` is a deprecated feature.

If you want to iterate over the indices, alternatives include:


```javascript
array.forEach((value, index) => { ... });
for (const [index, value] of array.entries()) { ... }
for (let i = 0; i < array.length; i++) { ... }
```

## Bad Practice

```javascript
for (const x in [3, 4, 5]) {
  console.log(x);
}
```

## Recommended

```javascript
for (const x in { a: 3, b: 4, c: 5 }) {
  console.log(x);
}
```
