# Found warning comments in code
**ID:** `JS-0099` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0099)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Documentation](https://img.shields.io/badge/type-documentation-lightgrey)

Developers often add comments to code that is not complete or needs review. You probably want to fix or review the code and then remove the comments before considering the code to be production-ready. Issues are raised whenever our analyzer finds these keywords in the comments i.e., TODO, FIXME, XXX.


```javascript
// TODO: do something
// FIXME: this is not a good idea
```

## Bad Practice

```javascript
// default - { "terms": ["todo", "fixme", "xxx"], "location": "start" }
function callback(err, results) {
  if (err) {
    console.error(err);
    return;
  }
  // TODO
}
```

## Recommended

```javascript
// default - { "terms": ["todo", "fixme", "xxx"], "location": "start" }
function callback(err, results) {
  if (err) {
    console.error(err);
    return;
  }
  // NOT READY FOR PRIME TIME
  // but too bad, it is not a predefined warning term
}
```
