# Prefer not to use words image, photo in image alt content
**ID:** `JS-0750` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0750)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Performance](https://img.shields.io/badge/type-performance-white)

Enforce img alt attribute does not contain the word image, picture, or photo. Screenreaders already announce img elements as an image. There is no need to use words such as image, photo, and/or picture.


## Bad Practice

```javascript
<img src="foo" alt="Photo of foo being weird." />
<img src="bar" alt="Image of me at a bar!" />
<img src="baz" alt="Picture of baz fixing a bug." />
```

## Recommended

```javascript
<img src="foo" alt="Foo eating a sandwich." />
<img src="bar" aria-hidden alt="Picture of me taking a photo of an image" /> // Will pass because it is hidden.
<img src="baz" alt={`Baz taking a ${photo}`} /> // This is valid since photo is a variable name.
```
