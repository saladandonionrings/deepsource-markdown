# Prefer consistent naming for boolean props
**ID:** `JS-0389` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0389)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Allows you to enforce a consistent naming pattern for props which expect a boolean value.


## Bad Practice

```javascript
// Example 1
var Hello = createReactClass({
  propTypes: {
    enabled: PropTypes.bool
  },
  render: function() { return <div />; };
});

// Example 2
type Props = {
  enabled: boolean
}
const Hello = (props: Props) => <div />;
```

## Recommended

```javascript
// Example 1
var Hello = createReactClass({
  propTypes: {
    isEnabled: PropTypes.bool
  },
  render: function() { return <div />; };
});

// Example 2
type Props = {
  isEnabled: boolean
}
const Hello = (props: Props) => <div />
```
