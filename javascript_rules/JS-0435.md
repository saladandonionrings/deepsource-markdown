# Avoid using `this.state` inside `this.setState()`
**ID:** `JS-0435` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0435)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

It is recommended not to use `this.state` inside `setState` calls. Such usage of `this.state` might result in errors because if two `setState` operations are called, they will be grouped into a single update, which will evaluate the old state instead of the current state.

This can be avoided by using callbacks that take the previous state as the first argument, then React will call the parameter with the correct and updated state, even when things happen in batches.


## Bad Practice

```javascript
function increment() {
  this.setState({value: this.state.value + 1});
}

this.setState({value: this.state.value + 1}) // 2
this.setState({value: this.state.value + 1}) // 2, not 3
```

## Recommended

```javascript
function increment() {
  this.setState(prevState => ({value: prevState.value + 1}));
}

this.setState({value: value + 1}) // 2
this.setState({value: value + 1}) // 3
```
