# Detected the use of classes as namespaces
**ID:** `JS-0327` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0327)

![Major](https://img.shields.io/badge/severity-major-orange)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Using class as a namespace can lead to errors.


## Bad Practice

```javascript
class EmptyClass {}

class ConstructorOnly {
  constructor() {
    foo();
  }
}

// Use an object instead:
class StaticOnly {
  static version = 42;
  static hello() {
    console.log('Hello, world!');
  }
}
```

## Recommended

```javascript
class EmptyClass extends SuperClass {}

class ParameterProperties {
  constructor(public name: string) {}
}

const StaticOnly = {
  version: 42,
  hello() {
    console.log('Hello, world!');
  },
};
```
