# Missing `providedIn` property
**ID:** `JS-0579` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0579)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Using the `providedIn` property makes classes decorated with `@Injectable` tree shakeable.

If you use `providedIn`, the injectable is registered as a provider of the module without adding it to the provider list of the module.


## Bad Practice

```javascript
@Injectable()
class Test {}
```

## Recommended

```javascript
@Injectable({
    providedIn: 'root'
})
class Test {}

@Injectable({
    providedIn: SomeModule
})
class Test {}
```
