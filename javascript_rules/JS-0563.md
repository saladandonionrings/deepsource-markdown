# Prefer to use `angular.isArray` instead of `typeof` comparisons
**ID:** `JS-0563` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0563)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

You should use the angular.isArray method instead of the default JavaScript implementation (typeof === "object Array").

**Rule based on Angular 1.x**


## Bad Practice

```javascript
// invalid
Object.prototype.toString.call(someArray) === '[object Array]'; // error: You should use the angular.isArray method

// invalid
Array.isArray(someArray) // error: You should use the angular.isArray method
```

## Recommended

```javascript
// valid
angular.isArray(someArray);
```
