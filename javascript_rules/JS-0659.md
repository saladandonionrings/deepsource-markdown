# Require control the display of the content inside `<transition>`
**ID:** `JS-0659` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0659)

![Major](https://img.shields.io/badge/severity-major-orange)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

After the release of Vue 3.0, the following changes have been made:

* Using`<transition>`as component's root will no longer trigger transitions when the component is toggled from the outside.* In 3.0, toggling a component with`<transition>`as its root node will no longer trigger the transition. Instead, the component should expose a boolean prop to control the presence of the content inside`<transition>`

## Bad Practice

```javascript
<transition><div /></transition>
<transition v-if="show">
  <div></div>
</transition>
```

## Recommended

```javascript
<transition><div v-if="show" /></transition>
<transition><div v-show="show" /></transition>
<transition>
  <div v-if="show"></div>
</transition>
```
