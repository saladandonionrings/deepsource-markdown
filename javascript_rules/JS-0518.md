# Ensure `controllerAs` syntax in routes or states
**ID:** `JS-0518` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0518)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

It is recommended to use Angular's `controllerAs` syntax when defining routes or states.

There are two ways to use `controllerAs` syntax:

* Using`controllerAs`key with`controller`name* Using`as`with`controller`name
`controllerAs` is syntactic sugar over `$scope`. You can still bind to the View and still access `$scope` methods. It helps avoid the temptation of using `$scope` methods inside a controller. It is better to avoid `$scope` or move the function to a factory and reference them from the controller.


## Bad Practice

```javascript
// Example: When missing controllerAs syntax
$routeProvider.when('/myroute', {
    controller: 'MyController'
})

// Example: controllerAs syntax is defined twice for the route "/myroute"

$routeProvider.when('/myroute', {
    controller: 'MyController as vm',
    controllerAs: 'vm'
})

Example: controllerAs syntax in state

$stateProvider.state('mystate', {
    controller: 'MyController'
})
```

## Recommended

```javascript
// Example: When missing controllerAs syntax
$routeProvider.when('/myroute', {
    controller: 'MyController',
    controllerAs: 'vm'
});

$routeProvider.when('/myroute', {
    controller: 'MyController as vm'
});

// Example: controllerAs syntax is defined twice for the route '/myroute'
$routeProvider.when('/myroute', {
    controller: 'MyController as vm'
});

$routeProvider.when('/myroute', {
    controller: 'MyController',
    controllerAs: 'vm'
});

// Example: controllerAs syntax in state
$stateProvider.state('mystate', {
    controller: 'MyController',
    controllerAs: 'vm'
})

$stateProvider.state('mystate2', {
    controller: 'MyController as vm'
})
```
