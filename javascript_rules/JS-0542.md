# Require and specify a prefix for all service names
**ID:** `JS-0542` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0542)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

All your services should have a name starting with the parameter you can define in your config object. The second parameter can be a Regexp wrapped in quotes. You can not prefix your services by "$" (reserved keyword for AngularJS services)

If the oldBehavior is true (default value), this rule will check the name of all services defined with the different methods provided by the framework : provider, service, factory, ... If this parameter is false, only services defined with the service method will be checked.

**Rule based on Angular 1.x**


## Bad Practice

```javascript
// valid
angular.module('myModule').service('prefixService', function () {
    // ...
});
```

## Recommended

```javascript
// invalid
angular.module('myModule').service('otherService', function () {
    // ...
}); // error: The otherService service should follow this pattern: /^xyz/
```
