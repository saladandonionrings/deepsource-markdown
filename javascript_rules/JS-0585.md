# `@Input` is renamed
**ID:** `JS-0585` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0585)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Two names for the same property i.e. one private, one public is inherently confusing.


## Bad Practice

```javascript
import { Input } from '@angular/core';
@Component({
    selector: 'foo'
})
class TestComponent {
    @Input('bar') set label(label: string) {}
}
```

```javascript
import { Input } from '@angular/core';
@Component({
    selector: 'foo'
})
class TestComponent {
    @Input('foo') label: string;
}
```

## Recommended

```javascript
import { Input } from '@angular/core';
@Component
class TestComponent {
    @Input() label: string;
}
```

```javascript
import { Input } from '@angular/core';
@Component
class TestComponent {
    @Input() set label(label: string) {}
}
```

```javascript
import { Input } from '@angular/core';
@Directive({
    selector: '[foo]'
})
class TestDirective {
    @Input('foo') bar = new EventEmitter<void>();
}
```
