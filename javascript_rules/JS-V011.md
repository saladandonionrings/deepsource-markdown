# Require template literals instead of string concatenation
**ID:** `JS-V011` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-V011)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

VueJS allows the ES6 template literals (`backticks`) to define string in readable manner.

ES6 template literals ("backticks") allow you to define your template across multiple lines, something you can't do in a regular JavaScript string. These are much easier to read and are supported now in many new browsers, though you should probably still transpile down to ES5 to be safe.


## Bad Practice

```javascript
<span>Message: {{ "Hello, " + name + "!" }}</span>
```

## Recommended

```javascript
<span>Message: {{ `Hello, ${name}!` }}</span>
```
