# Prefer to accompany `onClick` with some elements
**ID:** `JS-0746` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0746)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Enforce `onClick` is accompanied by at least one of the following: `onKeyUp` , `onKeyDown` , `onKeyPress` . Coding for the keyboard is important for users with physical disabilities who cannot use a mouse, AT compatibility, and screenreader users. This does not apply for interactive or hidden elements


## Bad Practice

```javascript
<div onClick={() => {}} />
```

## Recommended

```javascript
<div onClick={() => {}} onKeyDown={this.handleKeyDown} />
<div onClick={() => {}} onKeyUp={this.handleKeyUp} />
<div onClick={() => {}} onKeyPress={this.handleKeyPress} />
<button onClick={() => {}} />
<div onClick{() => {}} aria-hidden="true" />
```
