# Prevent explicit calls to lifecycle methods
**ID:** `JS-0573` | **Lien:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0573)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Explicit calls to lifecycle methods could be confusing. It is Angular's responsibility to invoke them.


## Bad Practice

```javascript
@Pipe({
  name: 'test'
})
class Test {
  test() {
    this.ngOnInit();
  }
}
```

```javascript
@Injectable()
class Test {
  test() {
    this.ngOnInit();
  }
}
```

## Recommended

```javascript
@Component({
  selector: 'test'
})

class Test {
  test() {
    super.ngAfterContentChecked();
  }
}
```

```javascript
@Component({
  selector: 'test'
})

class Test {
  test(): void {
    this.ngAfterContentChecked1();
    this.angAfterContentInit();
    this.ngAfterViewChecked2();
    this.ngAfterViewInit3();
    this.ngOnChange$();
    this.ngOnDestroyx();
    this.ngOnInitialize();
    this.ngDoChecking();
  }
}

@Directive({
  selector: 'test'
})

class TestDirective {
  test(): void {
    this.ngAfterContentChecked1();
    this.angAfterContentInit();
    this.ngAfterViewChecked2();
    this.ngAfterViewInit3();
    this.ngOnChange$();
    this.ngOnDestroyx();
    this.ngOnInitialize();
    this.ngDoChecking();
  }
}
```
