# Found confusing multiline expressions
**ID:** `JS-0024` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0024)

![Major](https://img.shields.io/badge/severity-major-orange)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

It is recommended to use a multiline expression with proper use of newlines; otherwise, it can cause confusion and have a different outcome than what the developer intended.

Consider the following code-snippet:


```javascript
someFuncCall()
[2].concat([3])
```
The two lines are being interpreted as one expression i.e


```javascript
someFuncCall()[2].concat([3])
```
This might even throw a runtime error if `someFuncCall()` does not return a multi-dimensional array.


## Bad Practice

```javascript
const upperLimit = limit
(1 || 2).get();

const hello = 'world'
[1, 2, 3].forEach(addNumber);

const x = function() {}
`hello`

const y = function() {}
y
`hello`

const z = upperLimit
/regex/g.test(limit)
```

## Recommended

```javascript
const upperLimit = limit;
(1 || 2).get();


// Two different expression separated using semi-colon
const upperLimit = limit
;(1 || 2).get()

const hello = 'world';
[1, 2, 3].forEach(addNumber);

const x = function() {};
`hello`

const tag = function() {}
tag `hello`
```
