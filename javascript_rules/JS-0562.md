# Prefer using `$timeout` instead of `setTimeout`
**ID:** `JS-0562` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0562)

![Minor](https://img.shields.io/badge/severity-minor-yellow)![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Instead of the default setTimeout function, you should use the AngularJS wrapper service $timeout.

**Rule based on Angular 1.x**


## Bad Practice

```javascript
// invalid
setTimeout(function() {
    // ...
}, 1000) // error: You should use the $timeout service instead of the default window.setTimeout method

// invalid
window.setTimeout(function() {
    // ...
}, 1000) // error: You should use the $timeout service instead of the default window.setTimeout method

// invalid
$window.setTimeout(function() {
    // ...
}, 1000) // error: You should use the $timeout service instead of the default window.setTimeout method
```

## Recommended

```javascript
// valid
$timeout(function() {
    // ...
}, 1000)
```
