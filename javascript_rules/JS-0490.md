# Avoid having unused expressions
**ID:** `JS-0490` | **Link:** [DeepSource](https://deepsource.com/directory/javascript/issues/JS-0490)

![Critical](https://img.shields.io/badge/severity-critical-red)![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

An unused expression that does not affect the state of the program indicates a logic error. It is recommended to avoid them as they affect the performance and also affect the program logic. For example, `n + 1;` is not a syntax error, but it might be a typing mistake where a programmer meant an assignment statement `n += 1;` instead. Sometimes, such unused expressions may be eliminated by some build tools in a production environment, which possibly breaks application logic. Type Cast expressions are ignored.


## Bad Practice

```javascript
someVar + 1
x?.y
```

## Recommended

```javascript
const val: number = someVar + 1
x?.y()
```
