# `as` with another context manager looks like a tuple
**ID:** `PYL-W0124` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-W0124)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

This `with` statement component returns multiple values and uses name binding with `as` only for a part of those values, as in `with ctx() as a, b` . This can be misleading since it's unclear if the context manager returns a tuple or if the node without a name binding is another context manager.


## Bad practice

```python
database = open('sqlite.db')

# Here we are entering two separate context managers, but it is unclear
with open('file.txt') as myfile, database:
    myfile.read()
```

## Recommended
One way to clarify this would be to use an `ExitStack` :


```python
from contextlib import ExitStack

database = open('sqlite.db')

with ExitStack() as stack:
    myfile = stack.enter_context(open('file.txt'))
    stack.enter_context(database)
```
