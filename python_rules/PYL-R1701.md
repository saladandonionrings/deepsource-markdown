# Consider merging `isinstance` calls
**ID:** `PYL-R1701` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-R1701)

![Major](https://img.shields.io/badge/severity-major-orange) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

You can pass a tuple of types you want to check as the second argument to `isinstance` . If the object matches with any of the types, it will return `True` else `False` .

It is therefore recommended to merge multiple consecutive `isinstance` calls into one. It is clearer and improves readability.


```python
# Instead of
if isinstance(i, int) or isinstance(i, float):
    print('A number')

# Do this
if isinstance(i, (int, float)):
    print('A number')
```
