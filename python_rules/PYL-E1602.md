# Parameter unpacking detected
**ID:** `PYL-E1602` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-E1602)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Tuple parameter unpacking has been removed from Python 3 and using it raises `SyntaxError` .

Tuple parameter unpacking is the use of a tuple as a parameter in a function signature so as to have a sequence argument automatically unpacked.

Consider the following code snippet when run using Python 2:


```python
def foo(a, (b, c), d):
    print a, b, c, d

t = (-5, 5)
foo(1, t, 3) ## Output: 1, -5, 5, 4
```
The tuple `t` is passed to `foo` . The tuple gets unpacked and `b` and `c` gets assigned the first and second element of tuple respectively. Running the code snippet using Python 3 will raise `SyntaxError` .


## Bad practice

```python
def some_function(a, (b, c), d):
    ...
```

## Recommended

```python
def some_function(a, b, c, d):
    ...
```
