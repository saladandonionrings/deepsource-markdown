# Name defined is both `nonlocal` and `global`
**ID:** `PYL-E0115` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-E0115)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

A name is being defined as `nonlocal` and `global` in the same scope. This will error out in the runtime as a name can't be both `nonlocal` and `global` .

The `nonlocal` statement causes the listed identifiers to refer to previously bound variables in the nearest enclosing scope excluding globals. It is to note here that the names listed in a nonlocal statement must not collide with pre-existing bindings in the local scope, which is being violated here.

Example:


```python
def func():
    x = "local"

    def nested():
        nonlocal x
        global x
        x = "nonlocal"
        ...
```
Here, x is `nonlocal` and `global` . Python will throw an error here `SyntaxError: name 'x' is nonlocal and global`
