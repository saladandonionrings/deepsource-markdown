# Setting loose POSIX file permissions is security-sensitive
**ID:** `PY-S6008` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PY-S6008)

![Major](https://img.shields.io/badge/severity-major-orange) ![Security](https://img.shields.io/badge/type-security-red)

Every file in the POSIX file system has the following permissions:

* Owner permissions - Determines what actions the owner of the file can perform on the file.
* Group permissions - Determines what actions a user who is a member of the group that a file belongs to can perform on the file.
* Other permissions − Determines what action all other users can perform on the file.

Granting permissions to `others` can lead to unintended access and modification to files. It is recommended to assign most restrictive permissions to files and directories.

One should avoid using following modes for a file unless necessary:

* [stat.S_IRWXO](https://docs.python.org/3/library/stat.html#stat.S_IRWXO) - Grants read and write permission to Other group.
* [stat.S_IWOTH](https://docs.python.org/3/library/stat.html#stat.S_IWOTH) - Grants write permission to Other group.
* [stat.S_IXOTH](https://docs.python.org/3/library/stat.html#stat.S_IXOTH) - Grants execute permission to Other group.

Refer to the references to learn about other modes for files.


## Bad practice

```python
import os, stat

r = os.umask(0) # read, write permission granted to others
os.umask(2)

os.chmod("/home/some_fle", stat.S_IRWXO)   # Others have read and write permission
os.lchmod("/home/some_file", stat.S_IRWXO)  # Others have read and write permission
os.fchmod(f, stat.S_IWOTH) # Others have write permission
```

## Recommended

```python
import os, stat

r = os.umask(0o027) # Others do not have read or write permission
os.umask(2)

os.chmod("/home/some_fle", stat.S_IRWXG)
os.lchmod("/home/some_file", stat.S_IRWXU)
os.fchmod(f, stat.S_IRWXG)
```
