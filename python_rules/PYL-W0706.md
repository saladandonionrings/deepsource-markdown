# Except handler raises immediately
**ID:** `PYL-W0706` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-W0706)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Using `raise` as the first or only operator of an except handler is useless because it raises back the exception immediately. Remove the `raise` operator or the entire `try-except-raise` block.


## Bad practice

```python
try:
    run_transaction(transaction_id)
except ValueError as e:
    raise TransactionError(e)
except:
    raise  # This is unnecessary
```

## Preferred

```python
try:
    run_transaction(transaction_id)
except ValueError as e:
    raise TransactionError(e)
```
