# Bad `type` operand placement
**ID:** `PTC-W0024` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PTC-W0024)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Style](https://img.shields.io/badge/type-style-blue)

The `type()` call shall be placed on the left side of the comparison when checking the type of one object against another type. This affects readability.


## Not preferred

```python
x = 42
if int is type(x):
    print('This is an integer')
```

## Preferred

```python
x = 42
if type(x) is int:
    print('This is an integer')
```
Or even better, use `isinstance` :


```python
x = 42
if isinstance(x, int):
    print('This is an integer')
```
