# Future import(s) name after other statements
**ID:** `PYL-W0410` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-W0410)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Imports from `__future__` should be the first imports in a file. If it's not, it will raise a `SyntaxError` .

In Python 2, dividing two integers results in an integer result. However, you can import `division` from the `__future__` module to enable float division everywhere. For it to consistently work, you need to ensure that it's the first statement in the file.


## Bad practice

```python
8 / 7  # Gives output: 1
...
from __future__ import division  # Will raise `SyntaxError`.
```

## Recommended

```python
from __future__ import division
...
8 / 7  # Gives output: 1.1428571428571428
```
