# Use of long suffix
**ID:** `PYL-E1606` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-E1606)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

`l` or `L` is used to mark a long integer. This will not work in Python 3, since `int` and `long` types have merged.


## Bad practice
In Python 2


```python
my_var = 1234l
```

## Recommended

```python
my_var = 1234
```
