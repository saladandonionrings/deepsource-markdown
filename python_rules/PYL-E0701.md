# Bad `except` order found
**ID:** `PYL-E0701` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-E0701)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Except clauses are not in the correct order (from the more specific to the more generic). Some exceptions will never reach the more specific error handlers because of this setup.


## Bad practice

```python
while True:
  try:
      int(input('Enter a number:'))
      break
  except BaseException:
      print('Invalid input')
  except KeyboardInterrupt:  # will never run, `BaseException` also catches `KeyboardInterrupt`
      pass
```

## Recommended
Order your exceptions from most specific to least specific. In this case, `KeyboardInterrupt` should appear before `BaseException` :


```python
while True:
  try:
      int(input('Enter a number:'))
      break
  except KeyboardInterrupt:
      pass
  except BaseException:
      print('Invalid input')
```
