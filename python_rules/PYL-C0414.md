# Import alias same as original package name
**ID:** `PYL-C0414` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-C0414)

![Major](https://img.shields.io/badge/severity-major-orange) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Import alias is same as original package name, thus making the alias unnecessary. This is probably not what's intended.


## Bad practice

```python
import my_package as my_package
```

## Recommended

```python
import my_package as mp
```
or, remove the import alias


```python
import my_package
```
Note: During autofix, DeepSource will remove the unnecessary alias.
