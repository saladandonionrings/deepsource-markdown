# Invalid operation on exceptions
**ID:** `PYL-W0716` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-W0716)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

An operation is being performed on an exception, which is not valid for the exception in question. Please note: Binary operations between exceptions in except handlers are not valid.


## Bad practice
Or operation is being performed on the exceptions here, which is an error:


```python
try:
    some_call()
except (ValueError | TypeError):
    pass
```

## Recommended
If the intention is to catch either of the two exceptions, a tuple would do the work:


```python
try:
    some_call()
except (ValueError, TypeError):
    pass
```
