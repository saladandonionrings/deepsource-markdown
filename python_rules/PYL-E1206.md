# Logging format string contains too few arguments
**ID:** `PYL-E1206` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-E1206)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Too few arguments are passed to the logging format string. Ensure that the number of arguments being passed to the logging call is the same as the number of arguments the format string can accept.


## Bad practice

```python
import logging
logging.info('LOG: %s (%s)', "E100")  # Expected 2 arguments, got 1
```

## Recommended

```python
import logging
logging.info('LOG: %s (%s)', "Missing data", "E100")
```
