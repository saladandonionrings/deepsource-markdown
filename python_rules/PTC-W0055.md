# Set size changed during iteration
**ID:** `PTC-W0055` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PTC-W0055)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Sets are represented by a hash table and adding or removing items while iterating over it will alter the iteration order. This will cause a `RuntimeError` .

If you need to modify the set during iteration, it is recommended to iterate over a [shallow copy](( [https://docs.python.org/3/library/copy.html](https://docs.python.org/3/library/copy.html) of the dictionary) of the set. A shallow copy creates a new object which stores the reference of the original elements.

For further reading, check

* [Shallow copy in Python](https://docs.python.org/3/library/copy.html)


## Bad practice

```python
magic_numbers = {0, 1, 1, 2, 3, 5}

for elem in magic_numbers:
    magic_numbers.add(get_next(elem))  # This will raise a `RuntimeError`.
```

## Recommended

```python
magic_numbers = {0, 1, 1, 2, 3, 5}

for elem in magic_numbers.copy():
    magic_numbers.add(get_next(elem))
```
