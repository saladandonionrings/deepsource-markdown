# The raise statement is not inside an except clause
**ID:** `PYL-E0704` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-E0704)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

A bare raise is used inside something which is not an `except` clause. This generates an error, since there are no active exceptions to be re-raised.


## Bad practice

```python
for num in numbers:
    if num == 0:
        raise  # No error has been caught to re-raise, so this doesn't work.

    inverse = 1 / num
```

## Recommended
The solution is to either only use a bare `raise` inside an `except` block, or to raise a new exception value:


```python
for num in numbers:
    if num == 0:
        raise ValueError('Expected non-zero value')  # Works just fine

    inverse = 1 / num
```
An exception to this rule is represented by a bare raise inside a `finally` clause, which might work, as long as an exception is raised inside the `try` block, but that means it won't always work, which is a code smell that must not be relied upon.
