# Unsafe parameter given to `subprocess.Popen`
**ID:** `PYL-W1509` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-W1509)

![Major](https://img.shields.io/badge/severity-major-orange) ![Security](https://img.shields.io/badge/type-security-red)

The `preexec_fn` parameter is not safe to use in the presence of threads in your application. The child process could deadlock before exec is called. If you must use it, keep it trivial. Minimize the number of libraries you call into. Ref: [https://docs.python.org/3/library/subprocess.html#popen-constructor](https://docs.python.org/3/library/subprocess.html#popen-constructor)
