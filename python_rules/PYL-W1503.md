# Constant passed to unittest `assert<Bool>`
**ID:** `PYL-W1503` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-W1503)

![Major](https://img.shields.io/badge/severity-major-orange) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

The first argument of `assertTrue` and `assertFalse` should be a condition. If a constant is passed as a parameter, the condition will be always `True` , making the test meaningless.


## Bad practice

```python
import unittest


class Tests(unittest.TestCase):
    def test_something(self):
        ''' Some test '''
        self.assertTrue('My String')
```

## Recommended

```python
import unittest


class Tests(unittest.TestCase):
    def test_something(self):
        ''' Simple test '''
        string = 'My String'
        self.assertTrue('My String' == string)
```
You also might accidentally use `assertTrue` instead of `assertEqual` , which also causes this issue to be raised.


```python
import unittest


class Tests(unittest.TestCase):
    def test_something(self):
        ''' Simple test '''
        some_var = 'It should be assertEqual'
        self.assertTrue('I meant assertEqual not assertTrue', some_var)
```

```python
import unittest


class Tests(unittest.TestCase):
    def test_something(self):
        ''' Simple test '''
        some_var = 'It should be assertEqual'
        self.assertEqual('I meant assertEqual not assertTrue', some_var)
```
