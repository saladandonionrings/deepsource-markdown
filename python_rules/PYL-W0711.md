# Exception to catch is the result of a binary operation
**ID:** `PYL-W0711` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-W0711)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

If an except clause needs to catch multiple exceptions, they must be expressed as a parenthesized tuple, for example: `except (ValueError, TypeError)` You can look at the docs [here](https://docs.python.org/3/tutorial/errors.html#handling-exceptions) . When `or` is used, only the first operand exception type of the conditional statement will be caught.


```python
In [1]: class Exc1(Exception):
    ...:     pass
    ...:
In [2]: class Exc2(Exception):
    ...:     pass
    ...:
In [3]: try:
    ...:     raise Exc1()
    ...: except Exc1 or Exc2:
    ...:     print("caught!")
    ...:
    caught!
In [4]: try:
    ...:     raise Exc2()
    ...: except Exc1 or Exc2:
    ...:     print("caught!")
    ...:
```
