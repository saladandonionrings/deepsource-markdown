# Ambiguous augmented assignment
**ID:** `PTC-W0061` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PTC-W0061)

![Minor](https://img.shields.io/badge/severity-minor-yellow) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

An augmented assign statement (such as `x += y` ) is performed on the same assign variable, which looks ambiguous here. Please verify if this was intended.

The augmented assignment has the same operand on the left and right-hand sides. This mostly looks like a bug due to an unnoticed typo. If this is intentional, it can be refactored to use multiple self-assignment, to make the intention clear.


## Bad practice

```python
myvar += myvar + 1
```

## Recommended

```python
# If it was a typo:
myvar += 1

# If this was intended:
# if the data you're dealing with is a number,
# consider using basic mathematical operation for better readability:
myvar = (2 * myvar) + 1
```
