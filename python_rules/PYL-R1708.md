# `StopIteration` detected in a generator
**ID:** `PYL-R1708` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-R1708)

![Major](https://img.shields.io/badge/severity-major-orange) ![Style](https://img.shields.io/badge/type-style-blue)

According to [PEP479](https://www.python.org/dev/peps/pep-0479/) , the raise of `StopIteration` to end the loop of a generator may lead to hard to find bugs. This PEP specifies that `raise StopIteration` has to be replaced by a simple `return` statement.


## Bad practice

```python
def some_generator(n):
    for i in range(n):
        yield i
        if i >= 5:
            raise StopIteration  # Will raise an exception
```

## Recommended

```python
def some_generator(n):
    for i in range(n):
        yield i
        if i >= 5:
            return  # Will close the generator
```
