# Class variable conflicts with slots
**ID:** `PYL-E0242` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-E0242)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

`__slots__` contain a name which is same as class variable, thus raising a conflict. During execution, Python will raise a `ValueError` complaining of the conflict.

`__slots__` is a special attribute to explicitly declare data members (like properties) and deny the creation of `__dict__` and `__weakref__` (unless explicitly declared in `__slots__` or available in a parent).

If the name of any slot variable conflicts with any class variable, it is recommended to rename or remove it.


## Bad practice

```python
class C:
    some_class_var = "Foo"
    __slots__ = ["some_class_var", "voila"]

    ...
```

## Recommended

```python
class C:
    some_class_var = "Foo"
    __slots__ = ["voila"]

    ...
```
