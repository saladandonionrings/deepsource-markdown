# Sequence value overwritten unconditionally
**ID:** `PTC-W0057` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PTC-W0057)

![Major](https://img.shields.io/badge/severity-major-orange) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

You are assigning values at the same sequence index continuously without performing any other operations between them. This might be something that was not intended and we suggest you retake a look at it. If you want the value of the later assignment to be stored, it is recommended to remove this assignment, since it is anyways going to be overwritten.


## Not preferred:

```python
some_numbers = [1, 2, 3, 4, 5]
some_numbers[0] = 3
some_numbers[0] = 4  # unconditionally overwritting the existing value

mymap = {'one': 1, 'two': 2}
mymap['one'] = 4
mymap['one'] = 3 # unconditionally overwritting value mapped to `one`
```

## Preferred:

```python
some_numbers = [1, 2, 3, 4, 5]
some_numbers[0] = 3
check_some_numbers()
some_numbers[0] = 4

mymap = {'one': 1, 'two': 2}
mymap['one'] = 4
if condition:
    mymap['one'] = 3
```
