# Multiple values found for keyword argument
**ID:** `PYL-E1132` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-E1132)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

A function call contains multiple values for the same keyword argument. This results in a `TypeError` as Python does not allow duplicate keyword arguments in function calls.

This error occurs when:

* The same keyword argument is specified multiple times in a function call
* A keyword argument is provided both positionally and as a keyword
* Dictionary unpacking with `**` results in duplicate keys

Here are examples of incorrect function calls:


```python
# Same keyword argument specified twice
def greet(name, age):
    print(f"{name} is {age} years old")

greet(name="Alice", name="Bob")  # TypeError: multiple values for argument 'name'

# Argument provided both positionally and as keyword
greet("Alice", name="Bob")  # TypeError: multiple values for argument 'name'

# Duplicate keys when unpacking dictionaries
user1 = {"name": "Alice", "age": 25}
user2 = {"name": "Bob", "age": 30}
greet(**user1, **user2)  # TypeError: multiple values for argument 'name'
```
To fix this, ensure that:

This error is particularly common when:

* Merging dictionaries for function calls
* Refactoring function calls with mixed positional and keyword arguments
* Working with automated argument generation or dynamic function calls
