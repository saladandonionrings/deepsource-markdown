# Unnecessary `elif` / `else` block after `continue`
**ID:** `PYL-R1724` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-R1724)

![Major](https://img.shields.io/badge/severity-major-orange) ![Style](https://img.shields.io/badge/type-style-blue)

The use of `else` or `elif` becomes redundant and can be dropped if the last statement under the leading `if` / `elif` block is a `continue` statement. In the case of an `elif` after `continue` , it can be written as a separate `if` block. For `else` blocks after `continue` , the statements can be shifted out of `else` . Please refer to the examples below for reference.

Refactoring the code this way can improve code-readability and make it easier to maintain.


## Bad practice

```python
def classify_number(x):
    for num in range(x):
        if x % 2 == 0:
            continue
        else:
            print(f"{num} is Odd}")
```

## Preferred:

```python
def classify_number(x):
    for num in range(x):
        if x % 2 == 0:
            continue
        print(f"{num} is Odd}")
```
