# Use of an unneeded `not` detected
**ID:** `PYL-C0113` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-C0113)

![Major](https://img.shields.io/badge/severity-major-orange) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

The boolean expression contains an unneeded negation and can be re-written without using the `not` statement.


## Bad practice

```python
def categorize_number(num):
    if not num % 2 == 0:
        print("The number is odd.")
    else:
        print("The number is even.")
```
Here, the condition `not num % 2 == 0` contains an unneeded `not` and can be changed into `num % 2 != 0` . A clearer and more straightforward approach would look like this:


## Recommended

```python
def categorize_number(num):
    if num % 2 != 0:
        print("The number is odd.")
    else:
        print("The number is even.")
```
