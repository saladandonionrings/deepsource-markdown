# Bad reversed sequence
**ID:** `PYL-E0111` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-E0111)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

The first argument to the `reversed()` builtin should be a sequence and the object type of the sequence should implement `__reversed__` , `__len__` and `__getitem__` . In the issue raised, the argument is violating one or more of the condition mentioned above. Python will throw a `TypeError` during execution.

Example:


```python
seq = reversed(None)  # Error here.
bad_reversed = reversed({2, 4, 6, 8})  # Sets are not reversible too. Error here.
```
