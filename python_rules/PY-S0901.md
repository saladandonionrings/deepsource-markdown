# Insecure use of `format_html` detected
**ID:** `PY-S0901` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PY-S0901)

![Major](https://img.shields.io/badge/severity-major-orange) ![Security](https://img.shields.io/badge/type-security-red)

Django's `format_html()` function can be used to safely insert untrusted user data into HTML.

However, passing an already formatting string to `format_html()` has no effect on the inputted string, and may be a security issue. This may expose [cross-site scripting (XSS)](https://owasp.org/www-project-top-ten/2017/A7_2017-Cross-Site_Scripting_(XSS)) vulnerabilities.


## Bad practice

```python
format_html(f"<b>{user_input}</b>")  # `user_input` is not being sanitized!
```

## Recommended

```python
format_html("<b>{}</b>", user_input)  # This can safely be used.
```
