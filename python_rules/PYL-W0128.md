# Variable redeclared during assignment
**ID:** `PYL-W0128` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-W0128)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

A variable is being redeclared in the same assignment. In this case, the variable will contain the last value assigned to it.

In most cases, the variable re-assignment is a result of a typing mistake, which can lead to unintentional results during the runtime. It is therefore recommended to take another look at the assignments highlighted in the issues.


## Bad practice

```python
for x, y, x in zip(range(1, 5), range(5, 10), range(15, 20)):  # Variable x is being redeclared.
     ...
```

## Recommended
Consider giving the variable a new name.


```python
for x, y, z in zip(range(1, 5), range(5, 10), range(15, 20)):
     ...
```
