# `assert` called on tuple
**ID:** `PYL-W0199` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-W0199)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

A call of `assert` on a tuple will always evaluate to `True` if the tuple is not empty, and will always evaluate to `False` if it is. If you meant to pass the 2nd argument as a message, it is recommended to remove the parenthesis around your assert values.


## Bad practice

```python
def test_abc():
    a = 1
    assert (a != 1, "Some message")
```

## Recommended

```python
def test_abc():
    a = 1
    assert a != 1, "Some message"
```
