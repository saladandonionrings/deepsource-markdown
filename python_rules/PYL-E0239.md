# Inheritance is not from a class
**ID:** `PYL-E0239` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-E0239)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Python classes can only inherit from a class. Inheriting from any other object is an error.


## Bad practice

```python
class MyClass:
    pass

my_obj = MyClass()

class SubClass(my_obj):  # Not a class, will raise an error
    pass
```

## Recommended

```python
class MyClass:
    pass

my_obj = MyClass()

class SubClass(MyClass):  # Extending a class
    pass
```

## Exceptions
If a class used as the base is dynamically generated and returned by a function, the checker may not correctly understand its type at the moment. Therefore, it is recommended to prevent DeepSource from raising this issue again using a `# skipcq` pragma.
