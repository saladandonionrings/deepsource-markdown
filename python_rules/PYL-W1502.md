# `datetime.time` used in a boolean context
**ID:** `PYL-W1502` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-W1502)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Using `datetime.time` in a boolean context can hide subtle bugs when the time they represent matches midnight UTC.

Midnight is represented by `datetime.time(0, 0, 0)` . However, this time (unlike all other valid times, including `datetime.time(0,0,1)` ) evaluates to `false` in `if` conditions for Python 3.4 and older versions.


## Bad practice

```python
import datetime

start_time = datetime.time(0, 0, 0)
if start_time:  # Evaluates to False
    doSomething()
```

## Recommended

```python
import datetime

start_time = datetime.time(0, 0, 0)
if start_time is not None:  # Evaluates to True
    doSomething()
```
