# Exception might be lost
**ID:** `PYL-W0150` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-W0150)

![Major](https://img.shields.io/badge/severity-major-orange) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

The `finally` block may swallow any exceptions raised in the `try` block. This can pose a bug risk.


## Bad practice

```python
def some_function():
    try:
        with open('myfile.txt') as file:
            data = file.read()
    except FileNotFoundError:
        print(fnf_error)
    finally:
        return data
```

## Recommended

```python
def some_function():
    try:
        with open('myfile.txt') as file:
            data = file.read()
    except FileNotFoundError:
        print(fnf_error)

    return data
```
