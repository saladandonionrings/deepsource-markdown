# not-async-context-manager
**ID:** `PYL-E1701` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-E1701)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

Async context manager is being used with an object that does not implement the async context management protocol ( `__aenter__` and `__aexit__` ). This will raise `AttributeError` during runtime since the attributes `__aexit__` and `__aenter__` are missing.

You can use following decorators to make an async context manager object:

* For Python 3.7 and above: `asynccontextmanager` decorator from `contextlib` .
* For Python 3.6: `asynccontextmanager` decorator from `async_genetator` .
* For Python 3.5: You'd need to use both `asynccontextmanager` and `async_generator` from the package `async_generator` .


## Bad practice

```python
class ContextManager:
    def __enter__(self):
        pass

    def __exit__(self, *exc):
        pass


async def foo():
    async with ContextManager():
        pass
```

## Recommended

```python
class AsyncContextManager:
    def __aenter__(self):
        pass

    def __aexit__(self, *exc):
        pass


async def foo():
    async with AsyncContextManager():
        pass
```
