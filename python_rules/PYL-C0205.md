# Slots are recommended to be an iterable, simple string detected
**ID:** `PYL-C0205` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-C0205)

![Major](https://img.shields.io/badge/severity-major-orange) ![Style](https://img.shields.io/badge/type-style-blue)

While it's technically correct to have simple strings as slot values, it is recommended to use an iterable like a `tuple` or a `list` . It's very common to add more than one property names in the slots, and even if there's one property name at the present, having that as a one-member iterable is more clear than a string.

This is a convention and helps it easier to understand and extend your code.


## Not preferred

```python
class Example:
    __slots__ = 'my_attr'
```

## Preferred

```python
class Example:
    __slots__ = ['my_attr']
```
