# `return` found in `__init__`
**ID:** `PYL-E0101` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PYL-E0101)

![Critical](https://img.shields.io/badge/severity-critical-red) ![Bug Risk](https://img.shields.io/badge/type-bug_risk-green)

The `__init__()` method is required to return `None` , but an explicit return value in its body was detected. Returning a value other than `None` will raise a `TypeError` . Even if the return value is `None` explicit return statement shall always be avoided.


## Bad practice

```python
class NotCorrect:
    def __init__(self):
        self.attr = 100
        return self.attr


class ExplicitReturn:
    def __init__(self):
        self.attr = 100
        return None
```

## Recommended

```python
class Correct:
    def __init__(self):
        self.attr = 100


class NoReturn:
    def __init__(self):
        self.attr = 100
```
