# Special method should return `NotImplemented`
**ID:** `PTC-W0054` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PTC-W0054)

![Major](https://img.shields.io/badge/severity-major-orange) ![Anti-pattern](https://img.shields.io/badge/type-anti_pattern-purple)

Special methods corresponding to numeric operators, rich comparison operators, and the `__length_hint__` method should return `NotImplemented` when the operation is not supported. These methods should not raise `NotImplementedError` as callers don't expect it and won't catch this exception.

For example, `A + B` is equivalent to calling `A.__add__(B)` . If this binary operation is not supported by the class `A` , `A.__add__(B)` should return `NotImplemented` . The interpreter will then try the reverse operation, i.e. `B.__radd__(A)` . This enables adding new operations by changing only one class instead of two.

For further reading, check these:

* Python documentation - [Built-in Constants - NotImplemented](https://docs.python.org/3/library/constants.html#NotImplemented)
* Python documentation - [Implementing the arithmetic operations](https://docs.python.org/3/library/numbers.html#implementing-the-arithmetic-operations)


## Not preferred:

```python
class A:
    def __init__(self, val): 
        self.val = val 
    
    def __add__(self, other): 
        raise NotImplementedError
         
class B: 
    def __init__(self, val): 
        self.val = val 
        
    def __add__(self, other): 
        return self.val + other.val 
     
    def __radd__(self, other): 
        return self.val + other.val
                
A(1) + B(1)  # This will raise `NotImplementedError`
```

## Preferred:

```python
class A:
    def __init__(self, val): 
        self.val = val 
    
    def __add__(self, other): 
        return NotImplemented
         
class B: 
    def __init__(self, val): 
        self.val = val 
        
    def __add__(self, other): 
        return self.val + other.val 
     
    def __radd__(self, other): 
        return self.val + other.val

A(1) + B(1)  # This will return 2
```
