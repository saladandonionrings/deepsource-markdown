# Audit required: Server certificate may not be verified
**ID:** `PTC-W6001` | **Lien:** [DeepSource](https://deepsource.com/directory/python/issues/PTC-W6001)

![Major](https://img.shields.io/badge/severity-major-orange) ![Security](https://img.shields.io/badge/type-security-red)

It is recommended to validate server certificate during SSL/TLS sessions to protect it from man-in-the-middle attacks.

The certificate chain validation takes care of the following things:

* The certificate is issued by its parent Certificate Authority or the root CA trusted by the system.
* Each CA is allowed to issue certificates.
* Each certificate in the chain is not expired.


## Not preferred:

```python
import ssl

context = ssl.create_default_context()
ctx3.verify_mode = ssl.CERT_NONE  # Setting this won't verify the certificate
```

## Preferred:

```python
import ssl

context = ssl.create_default_context()
ctx3.verify_mode = ssl.CERT_REQUIRED
```
